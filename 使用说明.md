# 山海·云烟使用说明

山海·云烟是一款辅助写小说的软件。

你可以用它写长篇、写短篇、连续生成章节、把小说改成剧本，也可以整理角色、世界观、章节摘要和参考资料。

这份说明只写给普通用户看。里面出现的英文，大多是软件页面上的原名字，不需要理解原理，照着说明填写就可以。

## 一、打开软件

1. 解压你拿到的软件压缩包。
2. 进入软件文件夹。
3. 双击 `启动山海·云烟.bat`。
4. 等几秒钟，浏览器会自动打开。

如果浏览器没有自动打开，就手动打开浏览器，输入：

```text
http://localhost:5656
```

如果启动窗口显示了别的地址，就以启动窗口里的地址为准。

使用软件时，不要关闭启动窗口。关闭后，浏览器页面可能就不能用了。

## 二、先记住一个重要规则

软件里有好几个创作模式。每个模式都有自己的用途，也有自己的数据。

请不要把它们当成同一个写作空间。

尤其要记住：

- 多Agent创作的数据，不会自动跑到无限续写里。
- 无限续写的数据，也不会自动变成多Agent创作的数据。
- 短篇创作是短篇自己的流程，不等于长篇项目。
- 小说转剧本是剧本转换工作台，不等于正文写作区。

所以，长篇如果在“多Agent创作”里开始，就尽量在多Agent创作里完成设定、大纲和正文。

长篇如果在“无限续写”里从第一章开始写，就尽量在无限续写里一章章继续写。

不要在多Agent里做完设定和大纲，又跑去无限续写里直接接正文。这样容易出现资料接不上、章节列表不一致、上下文不完整的问题。

## 三、项目是什么

项目可以理解成“一本书的工作空间”。

一本小说最好建一个项目。这样角色、世界观、章节、资料不会和别的书混在一起。

创建项目：

1. 点击左上角当前项目名称。
2. 点击“新建小说项目”。
3. 输入项目名称。
4. 简单写一句介绍，也可以先不写。
5. 保存。

切换项目后，左侧资料、章节、知识中心内容都会跟着切换。

## 四、左侧按钮分别是干什么的

左侧最左边一列是主要入口。

### 项目概览

用来看当前项目的大概情况。

适合刚打开软件时确认：现在是不是打开了正确的书。

### 多Agent创作

主要面向长篇小说。

这里适合做一本长篇的完整创作流程：收集想法、做世界观、做角色、大纲、章节正文、润色、检查。

如果你想写几十章、上百章，并且希望设定和章节尽量连贯，就用这个模式。

### 无限续写

主要用来连续生成章节正文。

它不只是“续写已有正文”，也可以从第一章开始写。

如果你想少做拆分，直接给一个开头或灵感，然后让软件一章一章往下写，可以用这个模式。

注意：它和多Agent创作是独立数据，不要把它当成多Agent正文区。

### 短篇创作

用来写一篇完整短篇。

它会按“输入素材 → 生成方案 → 导语 → 大纲 → 正文 → 质检 → 复审 → 书名 → 成稿”的顺序走。

适合短篇、脑洞、练笔、投稿前初稿。

### 小说转剧本

用来把小说内容改成剧本形式。

适合把正文拆成场景、对白、动作和剧本段落。

### 知识中心

用来管理记忆、资料节点和知识工作台。

它更像一个资料整理处，可以查找、保存、编辑与项目有关的知识内容。

### 设置

用来配置模型、外观、知识库、备份、提示词、技能和文字替换规则。

第一次使用时，最重要的是先配置“全局API配置”。

### 关于

用来看软件说明和版本相关信息。

### 收缩/展开侧边栏

左下角的侧边栏按钮可以收起或展开左边导航，让写作区更宽。

## 五、多Agent创作怎么用

多Agent创作是长篇创作模式。

页面里的“Agent”，可以理解成不同分工的写作助手。

常见分工包括：

- 沟通助手：帮你整理需求，问清楚你想写什么。
- 世界观构建师：帮你做世界设定、势力、规则。
- 大纲规划师：帮你做故事结构和章节方向。
- 章节撰写师：根据大纲写正文。
- 文字润色师：改善文字表达。
- 质量评估师：检查逻辑、人物、剧情问题。

推荐流程：

1. 在当前项目里进入“多Agent创作”。
2. 先用右侧创作助手说清楚你想写的题材、主角、风格。
3. 让它整理世界观和角色。
4. 再让它生成大纲。
5. 再让它写章节正文。
6. 写完后让它润色和检查。

示例：

```text
我想写一本东方玄幻长篇，主角出身低微但很有执念，风格是压抑后爆发。请先帮我整理故事核心、主角目标和主要矛盾。
```

如果你决定用多Agent创作写长篇，就在这个模式里继续推进，不要中途换到无限续写去接正文。

## 六、无限续写怎么用

无限续写是独立的连续正文模式。

它适合：

- 从第一章开始连续写。
- 导入已有小说后继续往后写。
- 快速生成多章初稿。
- 先写出来，再慢慢修。

常见操作：

- 选择模型：选择这次续写要用哪个模型。
- 自定义模型名：列表里没有时，可以手动输入模型名。
- 阶段总结间隔：每隔几章自动总结一次剧情。
- 故事开头或灵感：第一次开始写时，在这里输入开头、设定或灵感。
- 开始创作第一章：从当前输入开始生成第一章。
- 继续创作下一章：已有章节后，用它继续往后写。
- 纠正/补充要求：如果上一章方向不对，可以写修正意见再继续。
- 导入已有小说：可以导入 txt、md、docx。
- 导出：可以把无限续写里的章节导出为 txt、md、docx。
- 重置：清空当前无限续写内容，谨慎使用。
- 迁移到协作：把无限续写内容写入协作项目。这个是手动迁移，不代表两个模式一直互通。

从第一章开始写的示例：

```text
请从第一章开始写。主角叫林烬，出身破落小镇，第一章写他被迫离家，结尾留下悬念。
```

导入已有内容后续写的示例：

```text
请保持原有文风，从上一章结尾继续写下一章，重点写主角第一次发现家族秘密。
```

## 七、短篇创作怎么用

短篇创作是一套独立流程。

页面里的主要设置和按钮：

- 模型：选择短篇流程要用的模型。
- 创作素材输入框：可以输入题材、关键词、灵感、例文、人物想法。
- 目标字数：整篇短篇大概写多少字。
- 每章字数：每章大概写多少字。
- 识别素材并生成 3 个方案：让软件先给你几个故事方向。
- 生成导语：根据已选方案生成开头介绍。
- 生成大纲：生成短篇结构。
- 带意见重生成：大纲不满意时，写修改意见后重来。
- 生成本章：单独生成某一章。
- 一键生成全部正文：按大纲生成全部章节。
- 一键补全剩余正文：已经写了一部分时，继续补完剩下章节。
- 生成质检报告：检查正文问题。
- 采纳并进入复审：确认质检结果后进入下一步。
- 生成复审报告：再做一次通篇检查。
- 采纳并进入取名：确认后进入书名阶段。
- 生成 5 个书名：给出候选书名。
- 组装成稿并生成标签：整理成最终成稿。
- 导出 TXT / MD / DOCX：导出最终短篇。
- 清空当前短篇流程：重新开始，谨慎使用。

短篇建议一步步走，不要一开始就要求它直接写完所有东西。

比较好的输入：

```text
我想写一个雨夜旅馆悬疑短篇，关键词是旧相机、失踪房客、重逢，结尾要反转。请先给我 3 个故事方向。
```

## 八、小说转剧本怎么用

小说转剧本用于把小说内容改成剧本。

一般流程：

1. 进入“小说转剧本”。
2. 放入小说内容，或按页面提示导入文本。
3. 设置转换要求。
4. 生成剧本。
5. 检查场景、对白、人物动作是否满意。

如果结果仍然太像小说，可以这样补充：

```text
请减少心理描写，增加人物对白和动作，让它更像短剧剧本。
```

小说转剧本有自己的工作内容，不会自动变成长篇正文。

## 九、知识中心怎么用

知识中心分两类用途。

第一类是“记忆”。

你可以保存一些长期有用的信息，比如人物偏好、世界规则、写作习惯、重要设定。

常见操作：

- 全部记忆：查看所有记忆。
- 新建分类：给记忆分组。
- 启用/停用分类：决定这个分类是否参与使用。
- 删除分类：删除分类和里面内容，谨慎操作。
- 搜索：按关键词找记忆。
- 类型筛选：按偏好、设定、剧情等类型筛选。
- 只看启用条目：隐藏停用的内容。
- 每页数量：控制一次显示多少条。
- 新建条目：手动添加记忆。
- 保存条目：修改后保存。
- 删除条目：删除当前条目。
- 批量启用、批量停用、批量删除：对勾选内容一起操作。
- 清空当前筛选结果：删除当前筛选到的内容，谨慎使用。

第二类是“知识工作台”。

它适合搜索知识节点、查看相似节点、查看图谱关系。

常见操作：

- 搜索节点：按关键词找资料。
- 最近访问：查看刚看过的内容。
- 节点类型：按类型筛选。
- 保存节点：编辑节点后保存。
- 显示/隐藏图谱：查看内容之间的关系。

## 十、资料库和知识库不是一回事

这两个名字很像，但用途不同。

资料库：放角色、世界观、物品、事件线、大纲、章节、正文摘要这些写作资料。

知识库配置：决定软件怎么把内容整理成“方便搜索的资料”。它属于设置项。

普通理解：

- 你写进去、看得懂的设定，放在资料库。
- 软件后台为了更好搜索而整理出来的数据，由知识库配置管理。

## 十一、右侧创作助手怎么用

右侧创作助手可以像聊天一样使用。

你可以让它：

- 帮你想设定
- 帮你改人设
- 帮你检查剧情
- 帮你润色
- 帮你总结
- 帮你判断下一步写什么

示例：

```text
帮我检查主角动机有没有说服力。
```

```text
这一章节奏太慢，请指出哪些地方可以删。
```

```text
把这段改得更有压迫感，但不要改变剧情。
```

输入 `@` 可以引用项目里的角色、章节或设定。

比如：

```text
参考 @第一章 帮我设计第二章冲突。
```

## 十二、设置：外观主题

位置：设置 → 外观主题。

这里调整软件界面外观。

包含这些项目：

- 快捷主题：暗夜、浅白、护眼绿、暖光、深蓝。
- 色相：改变主色调。
- 饱和度：颜色鲜艳程度。
- 背景亮度：背景整体明暗。
- 文字亮度：文字整体明暗。
- 背景图片：选择一张图片作为背景。
- 清除背景：删除当前背景图片。
- 叠加层透明度：控制背景图片显示得清楚还是暗一些。
- 保存主题设置：保存当前外观。

普通用户建议：直接选一个快捷主题即可。

## 十三、设置：全局API配置

位置：设置 → 全局API配置。

这里配置“软件用哪个模型来写作”。

### 当前使用配置

包含这些项目：

- 选择API配置：选择你保存过的模型平台配置。
- 选择模型：选择当前要用的模型。
- 应用配置：把选择结果设为当前使用。
- 测试连接：检查当前配置能不能正常使用。
- 测试结果：会告诉你能不能用、哪里不对。

### 配置列表

这里可以新建、编辑、删除配置。

新建或编辑配置时会看到：

- 配置名称：给这套配置起个名字，比如“我的模型”。
- API 类型：选择连接方式。一般模型平台写“兼容 OpenAI”时，选 OpenAI Chat。
- API Base URL：模型平台给你的连接地址。
- API Key：模型平台给你的使用码。
- 模型列表：这套配置里可以放多个模型名。
- 添加：把手动输入的模型名加入列表。
- 获取：尝试从平台自动获取模型列表。
- 温度参数：控制写作发散程度。越高越自由，越低越稳。
- 最大输出长度：控制模型一次最多写多少内容。
- 保存：保存这套配置。

多个 Key 可以这样填：

```text
sk-第一个
sk-第二个
sk-第三个
```

也可以这样填：

```text
sk-第一个, sk-第二个；sk-第三个
```

保存后，软件会轮流使用这些 Key。

编辑已有配置时，如果 Key 输入框留空，就表示不修改原来的 Key。

### 全局超时设置

超时可以理解成“最多等多久”。

通用模型请求超时包括：

- 连接超时：连接模型平台最多等多久。
- 读取超时：等模型回复最多等多久。
- 写入超时：发送请求最多等多久。
- 连接池超时：排队等连接最多等多久。

短篇流程步骤超时包括：

- 导语生成
- 大纲生成
- 章节生成
- 质量检查
- 复审定稿
- 书名生成
- 标签生成

如果经常生成到一半失败，可以适当调大。普通用户不确定时，先保持默认。

## 十四、设置：知识库配置

位置：设置 → 知识库配置。

这里决定软件怎么整理和搜索项目内容。

### 向量化服务配置

页面上写“向量化”，你可以简单理解成：把资料整理成更容易被搜索和引用的样子。

包含这些项目：

- 嵌入模型来源：选择用线上模型，还是本地模型包。
- 硅基流动线上模型：使用硅基流动提供的线上能力。
- 本地模型包：使用软件本地带的模型包。

选择“硅基流动线上模型”时，会看到：

- 硅基流动 API Key：填写硅基流动的 Key。
- API Base URL：硅基流动连接地址。
- 嵌入模型：要使用的模型名称。
- 向量维度：模型返回的数据尺寸，普通用户不懂就别改。
- 测试连接：检查能不能连通。

选择“本地模型包”时，会看到：

- 选择文件：选择本地模型包。
- 安装模型包：安装你选择的模型包。
- 模型目录：本地模型所在位置。
- 模型文件：模型文件名。
- 池化方式：CLS 或 Mean。普通用户保持默认即可。
- 最大长度：一次处理文本的最大长度。普通用户保持默认即可。

### 章节摘要自动化

这个开关决定：写完章节后，是否自动生成章节摘要。

长篇建议开启。

它可以帮助后续写作记住前文发生了什么。

### 知识库数据统计

这里会显示：

- 章节数：已经整理进去的章节数量。
- 分块数：内容被拆成了多少小段。
- 向量数：可被搜索引用的数据数量。
- 存储 MB：占用空间大小。

如果下面出现章节列表，还可以：

- 全选：选中所有章节。
- 删除选中章节：删除这些章节的知识库数据。
- 清空当前项目所有知识库数据：删除当前项目的全部知识库整理数据。

注意：删除知识库数据，不等于删除你的小说正文，但会影响搜索和引用。仍然要谨慎。

### 检索参数

包含这些项目：

- 检索数量 Top-K：一次最多找回几条相关资料。
- 向量权重：更看重语义相似。
- 全文权重：更看重字面关键词。

普通用户建议保持默认。

### 文本分块设置

包含这些项目：

- 分块大小：每段资料大概切多长。
- 重叠大小：相邻资料段之间重复多少内容。

普通用户建议保持默认。

## 十五、设置：Agent配置

位置：设置 → Agent配置。

这里决定不同写作助手用哪个模型。

普通用户可以全部使用全局配置，不需要单独改。

页面包含：

- 高级 / 开发者模式：显示更多内部助手。普通用户建议关闭。
- 显示高级Agent：打开后能看到高级助手。
- 单独配置：给某一个助手单独指定模型。
- 选择API配置：这个助手使用哪套配置。
- 选择模型：这个助手使用哪个模型。
- 温度参数：这个助手写作时更稳还是更发散。
- 最大输出长度：这个助手一次最多输出多少内容。
- 测试连接：检查这个助手当前选择的配置能不能用。
- 保存所有Agent配置：保存本页所有修改。

常见核心助手包括：

- 沟通助手：负责理解你的需求。
- 世界观构建师：负责世界观。
- 大纲规划师：负责大纲。
- 章节撰写师：负责正文。
- 文字润色师：负责润色。
- 质量评估师：负责检查问题。
- 连续创作师：负责无限续写。
- 角色构建师：负责角色卡。
- 事件线构建师：负责主线、支线和人物线。
- 细纲构建师：负责更细的场景安排。
- 章纲构建师：负责单章目标和关键事件。

建议：只有当你明确知道某个助手应该用更强或更便宜的模型时，再打开“单独配置”。

## 十六、设置：备份管理

位置：设置 → 备份管理。

这里用来备份和恢复项目。

页面包含：

- 创建备份：立即备份当前项目。
- 导入备份：导入以前保存的备份文件。
- 自动备份设置：查看或进入自动备份相关设置。
- 启用自动备份：打开后自动保存备份。
- 自动备份状态：显示是否启用、下次备份时间。
- 备份列表：显示已有备份。
- 恢复：把项目恢复到某个备份状态，会覆盖当前项目内容，谨慎使用。
- 下载：把备份文件下载出来。
- 删除：删除某个备份。

建议：重要项目写到关键节点时，手动点一次“创建备份”。

## 十七、设置：提示词管理

位置：设置 → 提示词管理。

提示词可以理解成：每个写作助手做事前遵守的“工作说明”。

普通用户一般不需要改。

页面包含：

- 高级 / 开发者模式：显示更多内部提示词入口。普通用户建议关闭。
- 显示高级 Agent：打开后能看到更多助手。
- Agent 列表：选择要查看的助手。
- 系统提示词：这个助手的基本工作方式。
- 任务提示词：这个助手执行某类任务时的具体要求。
- 刷新：重新读取提示词。
- 保存：保存你改过的提示词。
- 恢复默认：把自定义内容恢复成软件默认内容。
- 模板变量说明：说明 `{novel_type}`、`{theme}`、`{chapter_number}` 这类内容会被软件自动替换。

注意：提示词改错可能会影响生成效果。不确定时不要改。

## 十八、设置：技能管理

位置：设置 → 技能管理。

技能可以理解成给助手增加的小能力。

页面包含：

- 可用技能：当前能用的技能列表。
- 启用：打开某个技能。
- 关闭：停用某个技能。
- 删除技能：删除某个技能，谨慎使用。
- 保存技能设置：保存启用/关闭状态。
- 触发方式：有些技能会提示怎么叫它出来。

常见技能可能包括联网搜索、热点搜索、写作辅助等。

普通用户建议：只开自己会用的技能，不认识的先别动。

## 十九、设置：正则替换规则

位置：设置 → 正则替换规则。

这个页面可以理解成“自动查找并替换文字”。

页面包含：

- 添加规则：新增一条替换规则。
- 启用：决定这条规则是否生效。
- 规则描述：写给自己看的说明。
- 匹配模式：要查找的文字规则。
- 替换为：找到后替换成什么。
- 删除规则：删除某条规则。
- 保存规则：保存当前规则列表。

普通用户如果不懂“正则”，可以把它当成高级替换工具。不确定时不要随便改。

## 二十、如果你看到资料库管理

有些页面会提供资料库管理入口。

它用来上传和管理参考文件。

常见项目包括：

- 统计信息：总文件数、文档数、图片数、总大小。
- 上传资料：选择文件并上传。
- 资料列表：查看已上传文件。
- 类型筛选：按文档、图片、参考资料、其他筛选。
- 搜索资料：按文件名搜索。
- 下载：下载某个资料。
- 删除：删除某个资料。

这些资料是给当前项目做参考用的。

## 二十一、推荐怎么选模式

如果你要写长篇，并且希望设定、大纲、正文都更完整：

用多Agent创作，并且一直在多Agent创作里推进。

如果你要从第一章开始快速连续写正文：

用无限续写，并且一直在无限续写里推进。

如果你要写短篇：

用短篇创作。

如果你要把已有小说改成剧本：

用小说转剧本。

如果你要整理资料、记忆和知识节点：

用知识中心。

## 二十二、常见问题

### 1. 测试连接失败

去“设置 → 全局API配置”检查：

- API Key 是否完整
- API Base URL 是否正确
- API 类型是否选对
- 模型名称是否正确
- 模型平台账号是否还有余额

### 2. 写出来的内容和设定对不上

先确认你是不是切错模式。

比如，多Agent创作里的资料不会自动进入无限续写。无限续写也不会自动读取多Agent的完整流程数据。

### 3. 多Agent和无限续写到底能不能混用

不建议混着用。

它们是两套创作方式，数据不自动互通。

如果你一定要把无限续写内容带到协作项目里，可以使用页面提供的“迁移到协作”。但这属于手动迁移，不是无缝接着写。

### 4. 短篇内容能不能直接接到长篇里

不建议直接接。

短篇创作是独立流程，适合完成一篇短故事。长篇请用多Agent创作或无限续写。

### 5. 知识库显示未配置

去“设置 → 知识库配置”，检查“嵌入模型来源”。

如果用本地模型包，选择“本地模型包”并保存。

如果用硅基流动线上模型，填写硅基流动 Key 后测试连接。

### 6. 生成太慢或总是中断

可以去“设置 → 全局API配置 → 全局超时设置”，适当调大等待时间。

短篇流程中断，也可以调大短篇各步骤的超时时间。

### 7. 不小心改乱了提示词或规则

提示词页面有“恢复默认”。

正则替换规则如果不确定，就先停用规则，再保存。

### 8. 担心项目丢失

去“设置 → 备份管理”，点击“创建备份”。

重要项目建议经常备份。

## 二十三、最省心的使用习惯

- 一本书建一个项目。
- 选好模式后，不要随便换模式接着写。
- 多Agent创作用来做完整长篇。
- 无限续写用来连续生成正文。
- 短篇创作用来写完整短故事。
- 小说转剧本只负责改剧本。
- 重要设定及时保存到资料库或知识中心。
- 设置里看不懂的高级项，保持默认。
- 重要节点及时创建备份。

你可以把山海·云烟当成一个写作工作台。

它能帮你写，但你要先选对模式，并在同一个模式里把事情做完。
