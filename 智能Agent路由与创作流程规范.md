# 智能Agent路由与创作流程规范

## 核心理念：智能路由 + 灵活流程

### 设计原则
1. **智能意图识别**：大模型分析用户输入，自动判断需要调用哪个Agent
2. **灵活流程调整**：根据用户需求动态调整执行顺序，不强制固定流程
3. **透明化执行**：每次调用Agent都明确告知用户原因和预期结果
4. **用户主导**：用户可以随时打断、跳过或重新执行某个阶段

---

## 一、Router Agent（智能路由器）- 系统核心

### 这是整个系统最重要的Agent！

```
用户输入 → Router分析意图 → 决定调用哪个Agent → 执行 → 返回结果
```

### Router的核心职责

#### 1. 意图识别与分类
Router必须能识别以下意图类型：

| 用户意图 | 调用的Agent | 示例输入 |
|---------|------------|---------|
| 需求收集 | Communicator | "我想写一部玄幻小说" |
| 构建世界观 | Worldbuilder | "帮我设计一个修仙世界" |
| 创建角色 | CharacterBuilder | "创建主角设定" |
| 规划大纲 | Outliner | "规划故事大纲" |
| 写作章节 | CreativeWriter | "写第5章" |
| 续写内容 | ContinuousWriter | "继续写下去" |
| 润色内容 | Polisher | "这段太平淡，帮我润色" |
| 质量检测 | QualityValidator | "检查有没有禁用词" |
| 综合评估 | Evaluator | "评估这章质量" |
| 扩写内容 | ContentExpansion | "字数不够，帮我扩写" |
| 搜索网络信息 | Skill: web_search | "帮我搜索修仙小说的常见设定" |
| 搜索热点梗 | Skill: trends_search | "最近有什么热梗可以用？" |
| 查询知识库 | Skill: knowledge_base | "主角现在什么境界？" |
| 查询文献资料 | Skill: web_search | "查一下唐朝的官职体系" |
| 普通对话 | 直接回复 | "你好" |

#### 2. Skill调用决策

Router不仅要决定调用哪个Agent，还要决定是否需要调用Skill工具：

**可用的Skill类型**：
- **web_search**：网络搜索（搜索引擎、百科等）
- **trends_search**：热点搜索（微博、知乎、B站等热榜）
- **knowledge_base**：知识库检索（项目内的设定、章节等）
- **file_operations**：文件操作（读取、保存、重命名等）

**Skill调用时机判断**：
```python
# 伪代码
def should_use_skill(user_input):
    # 搜索类关键词
    search_keywords = ["搜索", "查找", "查询", "找一下", "有没有", "什么是"]
    
    # 热点类关键词
    trend_keywords = ["热点", "热梗", "最近", "流行", "热搜", "话题"]
    
    # 知识库类关键词
    kb_keywords = ["主角", "角色", "设定", "世界观", "前文", "之前"]
    
    if any(kw in user_input for kw in trend_keywords):
        return "trends_search"
    elif any(kw in user_input for kw in kb_keywords):
        return "knowledge_base"
    elif any(kw in user_input for kw in search_keywords):
        return "web_search"
    else:
        return None  # 不需要Skill
```

#### 3. Agent与Skill的协同

**协同模式1：Skill增强Agent**
```
用户："写一章融入最近热点的内容"

流程：
  1. Router调用trends_search获取热点
  2. Router将热点数据传递给CreativeWriter
  3. CreativeWriter基于热点创作内容
```

**协同模式2：多Skill组合**
```
用户："写一章关于现代都市修仙的内容，要融入最近的热点"

流程：
  1. Router调用web_search搜索"都市修仙设定"
  2. Router调用trends_search获取热点
  3. Router将两者结合传递给CreativeWriter
  4. CreativeWriter综合创作
```

**协同模式3：Skill辅助决策**
```
用户："帮我写第5章"

流程：
  1. Router调用knowledge_base查询前4章内容
  2. Router检查是否有第5章大纲
  3. 如果信息充足，调用CreativeWriter
  4. 如果信息不足，提示用户补充
```

#### 3. 上下文依赖检查

Router在调用Agent前必须检查：

```python
# 伪代码示例
def check_dependencies(intent, context):
    dependencies = {
        "write_chapter": ["world", "outline", "previous_chapters"],
        "create_outline": ["world", "characters"],
        "create_character": ["world"],
        "polish_content": ["content"],  # 只需要内容本身
        "continue_writing": ["existing_content"],
    }
    
    required = dependencies.get(intent, [])
    missing = [dep for dep in required if not context.has(dep)]
    
    if missing:
        return {
            "can_proceed": False,
            "missing": missing,
            "suggestion": f"需要先完成：{', '.join(missing)}"
        }
    
    return {"can_proceed": True}
```

#### 4. Router的系统提示词要求

```markdown
## Router Agent 系统提示词

你是智能路由器，负责分析用户意图并调用合适的Agent。

### 核心职责
1. **意图识别**：准确判断用户想做什么
2. **依赖检查**：确认是否具备执行条件
3. **智能决策**：决定调用哪个Agent或如何回复
4. **透明沟通**：明确告知用户你的决策过程

### 决策流程
1. 分析用户输入，识别主要意图
2. 检查执行该意图所需的前置条件
3. 如果条件不足，先引导用户补充或调用前置Agent
4. 如果条件充足，调用对应的Agent
5. 如果是普通对话，直接友好回复

### 输出格式（强制）
每次决策都必须明确说明：

```
🔍 意图识别：[用户想做什么]
📊 条件检查：
   ✅ 已具备：[条件1]、[条件2]
   ❌ 缺少：[条件3]
   
💡 决策：[你的决策]
📞 调用Agent：[Agent名称]
🎯 执行目标：[具体要做什么]

⏭️ 预期结果：[用户会得到什么]
```

### 禁止行为
❌ 不要在条件不足时强行执行
❌ 不要不说明就调用Agent
❌ 不要只回复"收到"等无意义内容
❌ 不要忽略用户的明确指令

### 灵活性原则
- 用户可以跳过某些步骤（如：直接提供世界观，跳过构建）
- 用户可以只执行某个单独步骤（如：只润色一段文字）
- 用户可以打断流程，随时切换任务
- 用户可以重新执行某个步骤（如：重写某章）
```

## 二、参考创作流程（非强制，可灵活调整）

### 完整流程（适用于从零开始创作）

```
1. 需求收集（Communicator）
   ↓
2. 世界观构建（Worldbuilder）
   ↓
3. 角色设定（CharacterBuilder）
   ↓
4. 故事大纲（Outliner）
   ↓
5. 细纲规划（Outliner）
   ↓
--- 以下为每章循环 ---
6. 章纲制定（ContextStrategy）
   ↓
7. 上下文读取（ContentReader）
   ↓
8. 记忆检索（AuxMemory）
   ↓
9. 正文创作（CreativeWriter）
   ↓
10. 质量检测（QualityValidator）
   ↓
11. 综合评估（Evaluator）
   ↓
12. 内容润色（Polisher）← 如需要
   ↓
13. 字数扩写（ContentExpansion）← 如需要
   ↓
14. 文件保存（FileNaming）
   ↓
15. 上下文压缩（ContextCompressor）
   ↓
16. 一致性检测（Evaluator）
```

### 灵活场景示例

#### 场景1：用户只想润色一段文字
```
用户输入 → Router识别：润色意图
         → 直接调用Polisher
         → 返回润色结果
         
不需要：世界观、大纲、其他章节
```

#### 场景2：用户已有世界观，直接写大纲
```
用户输入 → Router识别：规划大纲
         → 检查：世界观已提供 ✅
         → 调用Outliner
         → 返回大纲
         
跳过：需求收集、世界观构建
```

#### 场景3：用户要续写现有章节
```
用户输入 → Router识别：续写意图
         → 检查：已有内容 ✅
         → 调用ContinuousWriter
         → 返回续写内容
         
不需要：大纲、前置章节
```

#### 场景4：用户要检查质量
```
用户输入 → Router识别：质量检测
         → 检查：内容已提供 ✅
         → 调用QualityValidator
         → 返回检测报告
         
不需要：其他任何前置条件
```

## 四、关键Agent的智能行为

### 1. Communicator（沟通助手）

**智能行为**：
- 不要一次性问太多问题，根据用户回答逐步深入
- 如果用户已经提供了某些信息，不要重复询问
- 如果用户说"随便"、"你决定"，提供2-3个具体建议供选择
- 识别用户的隐含需求（如：提到"爽文"→推断节奏要快）

**输出示例**：
```
📋 需求收集进度：40%

已了解：
✅ 小说类型：玄幻
✅ 主题：修仙逆袭

还需确认：
❓ 主角性格：热血型？冷静型？搞笑型？
   💡 建议：热血型适合快节奏爽文

⏭️ 确认后将调用Worldbuilder构建世界观
```

### 2. CreativeWriter（创意写作师）

**智能行为**：
- 实时显示字数进度
- 自动检测是否违反约束（禁用词、降智特征）
- 如果字数不足，自动标记需要扩写
- 如果质量不佳，自动标记需要润色

**输出示例**：
```
✍️ 正在创作第5章...
📊 创作进度：
   🔄 当前字数：1456/2000字（73%）
   ✅ 开场场景：完成
   ✅ 冲突展开：完成
   🔄 高潮部分：进行中...

⚠️ 实时检查：
   ✅ 禁用词：0个
   ✅ 降智特征：未检测到
   ✅ 逻辑连贯：正常

⏭️ 预计还需2分钟完成
```

### 3. QualityValidator（质量验证师）

**智能行为**：
- 详细列出所有检测项和结果
- 对于发现的问题，给出具体位置和修改建议
- 自动判断是否需要润色或重写

**输出示例**：
```
🔍 质量检测报告

📊 检测结果：
   ✅ 字数：2156字（达标）
   ✅ 禁用词：0个违规
   ⚠️ 降智特征：发现1处
      - 位置：第3段
      - 问题：连续使用3个"似乎"
      - 建议：替换为更明确的表达
   ✅ 逻辑连贯：通过

💡 综合建议：
   建议进行轻度润色，优化第3段的表达

⏭️ 是否调用Polisher进行润色？
```
