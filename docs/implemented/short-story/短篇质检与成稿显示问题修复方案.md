# 短篇质检与成稿显示问题修复方案

## 问题总结

根据用户截图反馈，存在以下问题：

### 1. 质检和复审后没有重新生成正文
**现象**：质检和复审报告生成后，没有按照要求重新生成有问题的章节

**原因**：
- 质检和复审的提示词明确要求"不要重新生成完整正文"（这是正确的，避免超时）
- 但前端代码期望从报告中提取修订后的章节（`parse_chapters_from_full_text`）
- 这两者存在逻辑冲突

**解决方案**：
1. 保持质检/复审只输出报告的设计（已优化，避免超时）
2. 前端在"采纳并进入下一阶段"时，如果报告指出有问题章节，应该：
   - 提示用户手动修改有问题的章节
   - 或者提供"重新生成有问题章节"的按钮
   - 而不是期望从报告中提取完整正文

### 2. 成稿组装时的显示问题
**现象**：编辑区显示了不应该显示的内容
- 书名显示书名号《》
- 导语显示"导语三（反转向）"等风格标注
- 正文显示章节标题（如"1. 重逢"）
- 显示MD格式符号（#、**等）
- 显示词条标签、内容标签、主分类等元数据

**原因**：
- `assemble_output` 方法生成的是**完整的Markdown格式成稿**，包含所有元数据
- 前端直接将这个完整成稿显示在编辑区
- 但用户期望的是**纯净的正文内容**，不包含格式和元数据

## 修复方案

### 方案A：新增纯净正文组装方法（推荐）⭐⭐⭐⭐⭐

在 `short_story_service.py` 中新增一个方法，专门用于生成纯净的正文内容：

```python
def assemble_clean_text(self, workflow: Dict[str, Any]) -> str:
    """组装纯净的正文内容（不含格式和元数据）"""
    if not workflow.get("selected_title"):
        raise ValueError("尚未选择书名")
    if not workflow.get("selected_synopsis"):
        raise ValueError("尚未选择导语")
    if not workflow.get("chapters"):
        raise ValueError("尚无正文内容可组装")

    title = workflow["selected_title"]
    synopsis = workflow["selected_synopsis"]
    
    # 去除导语中的风格标注（如"（反转向）"）
    clean_synopsis = re.sub(r'[（(][^）)]*[）)]', '', synopsis).strip()
    
    # 组装纯净正文
    chapter_blocks = []
    for chapter in workflow["chapters"]:
        # 只输出章节内容，不输出章节标题
        chapter_blocks.append(chapter['content'])
    
    clean_output = (
        f"{title}\n\n"  # 书名不加书名号
        f"{clean_synopsis}\n\n"  # 导语不含风格标注
        f"{chr(10).join(chapter_blocks)}\n\n"  # 正文不含章节标题
        f"（全文完）"
    )
    
    return clean_output
```

**前端调用**：
```javascript
// 在 short-story-render.js 的 renderShortStoryFinalView 中
// 使用 clean_output 而不是 final_output
<textarea>${escapeHtml(workflow?.clean_output || '')}</textarea>
```

### 方案B：修改现有 assemble_output 方法（不推荐）

直接修改 `assemble_output` 方法，但这会影响导出功能，因为导出需要完整的Markdown格式。

### 方案C：前端过滤显示（快速方案）⭐⭐⭐

在前端显示时，对 `final_output` 进行处理：

```javascript
function getCleanFinalOutput(workflow) {
    const finalOutput = workflow?.final_output || '';
    if (!finalOutput) return '';
    
    // 提取纯净内容
    let clean = finalOutput;
    
    // 1. 去除书名号
    clean = clean.replace(/《([^》]+)》/g, '$1');
    
    // 2. 去除Markdown标题符号
    clean = clean.replace(/^#+\s*/gm, '');
    
    // 3. 去除粗体标记
    clean = clean.replace(/\*\*([^*]+)\*\*/g, '$1');
    
    // 4. 去除元数据行（主分类、内容标签、词条标签）
    clean = clean.replace(/^\*\*主分类\*\*：.+$/gm, '');
    clean = clean.replace(/^\*\*内容标签\*\*：.+$/gm, '');
    clean = clean.replace(/^\*\*词条标签\*\*：.+$/gm, '');
    
    // 5. 去除分隔线
    clean = clean.replace(/^---+$/gm, '');
    
    // 6. 去除"## 导语"和"## 正文"标题
    clean = clean.replace(/^##\s*导语\s*$/gm, '');
    clean = clean.replace(/^##\s*正文\s*$/gm, '');
    
    // 7. 去除章节标题（如"### 1. 重逢"）
    clean = clean.replace(/^###\s*\d+\.\s*.+$/gm, '');
    
    // 8. 去除导语中的风格标注
    clean = clean.replace(/[（(][^）)]*向[）)]/g, '');
    
    // 9. 清理多余空行
    clean = clean.replace(/\n{3,}/g, '\n\n');
    
    return clean.trim();
}
```

## 预期效果

### 质检/复审
- 生成报告后，用户清楚知道哪些章节有问题
- 用户可以手动修改或重新生成有问题的章节
- 不再期望从报告中提取完整正文

### 成稿显示
- 编辑区只显示纯净内容：
  - 书名：`离婚后，前夫跪求我回家`（无书名号）
  - 导语：纯文本内容（无风格标注）
  - 正文：直接从第一句开始（无章节标题）
  - 无MD格式符号
  - 无元数据标签

### 导出功能
- 保持不变，继续导出完整格式的成稿
- TXT/MD/DOCX 格式都包含完整元数据
