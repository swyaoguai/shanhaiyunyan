# 短篇质检流程优化方案

## 当前问题

### 现有流程
1. 质量检查：一次性发送所有章节 → 超时（输入15000+ tokens）
2. 通篇复审：一次性发送所有章节 → 超时

### 已实施的优化

#### ✅ 质量检查（已采用组合C方案）
- **方案**：分批处理 + 流式输出 + 快速模型
- **效果**：成功率100%，6-9分钟完成
- **详情**：见 [`docs/组合C方案实施说明.md`](组合C方案实施说明.md)

#### ⏳ 通篇复审（待优化）
- **当前问题**：仍然一次性发送所有章节，存在超时风险
- **推荐方案**：采用与质量检查相同的组合C方案

## 通篇复审优化方案

### 推荐方案：复用组合C方案

通篇复审与质量检查类似，都面临输入token过多的问题。建议采用相同的组合C方案：

#### 方案特点
1. **分批处理**：每3章一批进行复审
2. **流式输出**：实时反馈进度
3. **快速模型**：使用非思考型模型
4. **智能合并**：自动合并各批次的复审结果

#### 实施步骤

**1. 添加分批复审提示词模板**

在 `novel_agent/short_story_service.py` 中添加：

```python
BATCH_COHERENCE_REVIEW_PROMPT_TEMPLATE = """你是一位资深的文学复审编辑。

【用户原始词条】
{keywords}

【主分类】
{category}

【选定导语】
{selected_synopsis}

【当前批次正文（第{batch_start}章到第{batch_end}章）】
{batch_chapters_text}

请快速进行通篇复审，重点审核：
1. 词条覆盖度：关键词条是否在正文中体现
2. 导语一致性：核心冲突、角色设定是否与导语一致
3. 主题统一性：全文主题和基调是否一致
4. 章节衔接：本批次章节之间的衔接是否自然

输出格式（简洁）：
- 如无问题：✅ 本批次复审通过。
- 如有问题：
  第X章：问题类型 - 简要描述（不超过20字）
  第Y章：问题类型 - 简要描述（不超过20字）

注意：
1. 只列出最严重的逻辑问题，每章最多1个问题
2. 总共不超过8行
3. 不要重新生成完整正文
4. 系统会根据你的建议进行后续处理"""
```

**2. 增强 `build_coherence_review_prompt` 方法**

```python
def build_coherence_review_prompt(
    self,
    workflow: Dict[str, Any],
    use_batch: bool = True,
    batch_size: int = 3
) -> Dict[str, Any]:
    """构建复审提示词，支持分批处理"""
    machine = ShortStoryWorkflowStateMachine(workflow)
    machine._assert_state([ShortStoryStage.COHERENCE_REVIEWING])
    state = machine.snapshot()
    
    chapters = state["chapters"]
    total_chapters = len(chapters)
    
    # 如果章节数少于等于batch_size，或者不使用分批，则使用完整复审
    if not use_batch or total_chapters <= batch_size:
        prompt = COHERENCE_REVIEW_PROMPT_TEMPLATE.format(
            keywords=self._format_keywords(state["keywords"]),
            category=state.get("category") or "其他",
            selected_synopsis=state["selected_synopsis"],
            revised_full_text=self.render_chapters(chapters),
        )
        return {
            "success": True,
            "data": {
                "workflow": state,
                "prompt": prompt,
                "use_batch": False,
                "total_batches": 1,
            }
        }
    
    # 分批处理：生成多个批次的提示词
    batches = []
    for i in range(0, total_chapters, batch_size):
        batch_chapters = chapters[i:i + batch_size]
        batch_start = batch_chapters[0]["chapter_number"]
        batch_end = batch_chapters[-1]["chapter_number"]
        
        batch_prompt = BATCH_COHERENCE_REVIEW_PROMPT_TEMPLATE.format(
            keywords=self._format_keywords(state["keywords"]),
            category=state.get("category") or "其他",
            selected_synopsis=state["selected_synopsis"],
            batch_chapters_text=self.render_chapters(batch_chapters),
            batch_start=batch_start,
            batch_end=batch_end,
        )
        
        batches.append({
            "batch_index": len(batches),
            "batch_start": batch_start,
            "batch_end": batch_end,
            "prompt": batch_prompt,
            "chapter_count": len(batch_chapters),
        })
    
    return {
        "success": True,
        "data": {
            "workflow": state,
            "use_batch": True,
            "batch_size": batch_size,
            "total_batches": len(batches),
            "batches": batches,
        }
    }
```

**3. 复用批处理辅助函数**

通篇复审可以直接使用质量检查的辅助函数：
- `run_batch_quality_check_stream` - 改名为通用的批处理函数
- `merge_batch_quality_reports` - 同样适用于复审报告合并

#### 使用示例

```python
from novel_agent.short_story_service import get_service

service = get_service()

# 1. 构建分批复审提示词
result = service.build_coherence_review_prompt(
    workflow=workflow_state,
    use_batch=True,
    batch_size=3
)

if result["data"]["use_batch"]:
    batches = result["data"]["batches"]
    
    # 2. 定义LLM调用函数
    async def llm_call_func(prompt):
        return await llm_client.call(
            messages=[{"role": "user", "content": prompt}],
            stream=True
        )
    
    # 3. 执行分批复审
    final_report = await service.run_batch_quality_check_stream(
        batches=batches,
        llm_call_func=llm_call_func
    )
    
    # 4. 记录复审结果
    passed = "✅" in final_report and "通过" in final_report
    service.record_coherence_review(
        workflow=workflow_state,
        report=final_report,
        passed=passed
    )
```

- 实施简单，只需修改提示词

- 降低超时风险

#### 缺点
- 检查不如分批详细
- 仍有一定超时风险

#### 前端调用流程

```javascript
// 质量检查阶段
async function performQualityCheck(workflow) {
    const chapters = workflow.chapters;
    
    for (let chapter of chapters) {
        // 1. 检查单章
        const checkResult = await api.qualityCheckChapter({
            workflow,
            chapter_number: chapter.chapter_number,
            chapter_content: chapter.content
        });
        
        // 2. 如有问题，重新生成
        if (checkResult.has_issues) {
            const revised = await api.regenerateChapterWithFeedback({
                workflow,
                chapter_number: chapter.chapter_number,
                original_content: chapter.content,
                quality_issues: checkResult.issues
            });
            
            // 3. 更新章节
            chapter.content = revised.content;
        }
    }
    
    return workflow;
}

// 通篇复审阶段
async function performCoherenceReview(workflow) {
    // 1. 整体检查
    const reviewResult = await api.coherenceReview({
        workflow
    });
    
    // 2. 针对有问题的章节重新生成
    for (let issue of reviewResult.chapter_issues) {
        const revised = await api.regenerateChapterWithFeedback({
            workflow,
            chapter_number: issue.chapter_number,
            original_content: getChapterContent(workflow, issue.chapter_number),
            quality_issues: [issue.description]
        });
        
        updateChapter(workflow, issue.chapter_number, revised.content);
    }
    
    return workflow;
}
```

### 方案B：修改现有API行为（快速方案）

保持API不变，但修改prompt和处理逻辑：

1. **质量检查prompt**：只输出问题报告（已完成）
2. **前端处理**：收到报告后，前端解析问题章节，逐章调用重新生成API
3. **复审同理**

#### 优点
- 不需要新增API
- 前端改动较小

#### 缺点
- 前端需要解析AI返回的问题报告（格式可能不稳定）
- 需要多次API调用
