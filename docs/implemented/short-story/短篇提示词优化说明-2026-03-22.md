# 短篇创作提示词优化说明

## 优化时间
2026-03-22

## 优化目标
1. 增强大纲的情绪节点标注（爽点/虐点/转折点）
2. 优化正文创作，减少环境描写，强化情绪和冲突
3. 修复章节信息显示问题

## 具体修改

### 1. 大纲提示词优化（OUTLINE_PROMPT_TEMPLATE）

#### 新增要求
- 第8条：**重点标注情绪节点**
  - 爽点（如：打脸、逆袭、反转、揭秘）
  - 虐点（如：误会、背叛、失去、痛苦）
  - 转折点（如：身份揭露、真相大白、关系逆转）
  - 情绪高潮（如：冲突爆发、情感宣泄、决战时刻）

#### 输出格式新增
```
### 1. {{标题}}
- 摘要：...
- 出场角色：...
- 核心事件：...
- 叙事功能：...
- 情绪节点：【爽点/虐点/转折点】具体描述
```

### 2. 正文提示词优化（CHAPTER_PROMPT_TEMPLATE）

#### 核心变化
1. **移除章节标题输出**
   - 旧：`本章标题：{chapter_heading}`
   - 新：直接从正文第一句开始写
   - 第10条明确：**不要输出章节标题**

2. **强化情绪节点遵循**
   - 第2条：**严格遵循本章大纲的情绪节点**（爽点/虐点/转折点），确保情绪张力到位

3. **以导语为核心**
   - 第3条：**以导语为核心**，所有情节推进必须围绕导语设定的核心冲突展开

4. **减少环境描写**
   - 第6条：**减少环境描写**，重点放在对话、动作、心理活动和情绪冲突上

5. **强化情绪拉扯**
   - 第7条：根据主分类特点，充分展现人物情绪起伏
     - 爽文：突出打脸、逆袭、碾压的快感
     - 虐文：深挖误会、痛苦、无奈的情绪
     - 甜宠：强化甜蜜互动、宠溺细节

### 3. 数据结构优化

#### chapter_blueprints 新增字段
```python
{
    "chapter_number": 1,
    "title": "标题",
    "summary": "摘要",
    "characters": "出场角色",
    "core_event": "核心事件",
    "narrative_function": "叙事功能",
    "emotion_point": "情绪节点"  # 新增
}
```

#### 相关函数修改
1. `_extract_chapter_blueprints()` - 添加情绪节点提取
2. `_normalize_chapter_blueprints()` - 添加情绪节点规范化
3. `build_chapter_prompt()` - 将情绪节点传递到正文提示词

### 4. 前端显示优化

#### short-story-render.js 修改
```javascript
<div class="short-story-chapter-meta">
    <div><strong>摘要：</strong>${escapeHtml(blueprint.summary || '待补充')}</div>
    <div><strong>出场角色：</strong>${escapeHtml(blueprint.characters || '待补充')}</div>
    <div><strong>核心事件：</strong>${escapeHtml(blueprint.core_event || '待补充')}</div>
    <div><strong>叙事功能：</strong>${escapeHtml(blueprint.narrative_function || '待补充')}</div>
    ${blueprint.emotion_point ? `<div><strong>情绪节点：</strong>${escapeHtml(blueprint.emotion_point)}</div>` : ''}
</div>
```

## 预期效果

### 大纲生成
- 每章会明确标注情绪节点类型和具体内容
- 便于作者和AI都清楚每章的情绪重点

### 正文创作
- 不再输出章节标题（避免重复）
- 更聚焦于情绪冲突和人物互动
- 减少冗长的环境描写
- 根据主分类特点强化对应情绪（爽/虐/甜）
- 严格围绕导语核心冲突展开

### 用户体验
- 章节信息显示更完整（包含情绪节点）
- 正文更符合网文读者的阅读习惯
- 情绪节奏更明确，创作方向更清晰

## 使用建议

1. **重新生成大纲**：已有项目建议重新生成大纲以获得情绪节点信息
2. **检查情绪节点**：生成大纲后，检查每章的情绪节点是否合理
3. **调整主分类**：确保主分类设置正确，以便AI生成对应风格的内容
4. **导语质量**：导语质量直接影响正文质量，建议精心选择导语

## 兼容性说明

- 旧版大纲（无情绪节点字段）仍可正常使用
- 前端会自动判断是否显示情绪节点
- 不影响已有的工作流程和数据结构
