
# 短篇创作 Skill 设计文档 --- ## 一、Skill 概述 | 项目 | 说明 | |------|------| | **Skill 名称** | `short_story_creator` | | **功能描述** | 基于用户提供的词条，自动完成从构思到成稿的短篇小说全流程创作 | | **输入** | 用户提供的关键词条（1~N 个） | | **最终输出** | 书名 + 词条标签 + 导语 + 完整正文 | --- ## 二、完整工作流（Pipeline） ``` ┌─────────┐ │ 用户输入词条 │ └────┬────┘ ▼ ┌──────────────────┐ │ Step 1: 导语生成 │ → 生成 5 条导语（各约200字），用户选择1条 └────┬─────────────┘ ▼ ┌──────────────────┐ │ Step 2: 大纲生成 │ → 基于选定导语，生成章节大纲 └────┬─────────────┘ ▼ ┌──────────────────┐ │ Step 3: 正文创作 │ → 逐章按大纲撰写正文 └────┬─────────────┘ ▼ ┌──────────────────┐ │ Step 4: 质量检查 │ → 字数 / 逻辑 / 时间线 / 角色一致性 └────┬─────────────┘ ▼ ┌──────────────────────┐ │ Step 5: 通篇复审 │ → 校验全文与导语、词条的契合度 └────┬─────────────────┘ ▼ ┌──────────────────┐ │ Step 6: 书名生成 │ → 生成 5 个候选书名，用户选择 └────┬─────────────┘ ▼ ┌──────────────────────┐ │ 最终输出：组装完整作品 │ │ 书名 + 词条标签 + 导语 + 正文 │ └──────────────────────┘ ``` --- ## 三、各步骤详细定义 ### Step 1：导语生成（Synopsis Generation） ```yaml step_id: generate_synopsis input: 用户词条列表 (keywords[]) output: 5 条候选导语 (synopsis[5]) interaction: 需要用户选择 rules: - 每条导语字数：约 200 字（180~220 字） - 5 条导语之间风格/方向应有差异化 - 例：悬疑向、温情向、反转向、暗黑向、治愈向 - 每条导语需包含：核心冲突、主要角色轮廓、故事基调 - 所有导语必须完整覆盖用户给出的全部词条 ``` **Prompt 模板参考：** ```text 你是一位专业的短篇小说策划编辑。 用户提供了以下创作词条： {keywords} 请根据这些词条，生成 5 条风格各异的故事导语。 要求： 1. 每条导语约 200 字（180~220字） 2. 5 条导语应覆盖不同的故事方向（如：悬疑、温情、反转、暗黑、治愈等） 3. 每条导语需包含：核心冲突、主要角色轮廓、整体基调 4. 所有词条必须在导语中得到体现 5. 导语应具有吸引力，让读者想要继续阅读 请按以下格式输出： 【导语一】（XX 向） {内容} 【导语二】（XX 向） {内容} ... 以此类推 ``` **用户交互：** ```text 请从以上 5 条导语中选择您最满意的一条（回复编号 1-5）， 或提出修改意见，我将为您调整。 ``` --- ### Step 2：大纲生成（Outline Generation） ```yaml step_id: generate_outline input: 选定的导语 (selected_synopsis) + 原始词条 (keywords[]) output: 章节大纲 (outline{}) interaction: 可选（用户可确认或调整） rules: - 根据故事体量自动规划章节数（建议 3~8 章） - 每章大纲包含： - 章节标题 - 本章摘要（100~150 字） - 出场角色 - 核心事件/冲突 - 本章在全篇中的叙事功能（铺垫/转折/高潮/收束等） - 需包含完整的角色表（姓名、身份、关系） - 需包含时间线梳理 ``` **Prompt 模板参考：** ```text 你是一位经验丰富的小说大纲策划师。 【创作词条】 {keywords} 【选定导语】 {selected_synopsis} 请根据以上信息，生成一份详细的短篇小说章节大纲。 要求： 1. 合理规划章节数量（3~8 章），每章正文约 1000 字 2. 每章需包含：章节标题、本章摘要（100-150字）、出场角色、核心事件、叙事功能 3. 在大纲开头列出： - 【角色表】：所有角色的姓名、身份、相互关系 - 【时间线】：故事的时间跨度与关键时间节点 4. 确保起承转合完整，叙事节奏合理 5. 情节须与导语描述保持一致 输出格式： ## 角色表 | 角色 | 身份 | 关系 | |------|------|------| ## 时间线 ... ## 章节大纲 ### 第一章：{标题} - 摘要：... - 出场角色：... - 核心事件：... - 叙事功能：... （以此类推） ``` --- ### Step 3：正文创作（Content Writing） ```yaml step_id: write_content input: 大纲 (outline) + 导语 (selected_synopsis) + 角色表 + 时间线 output: 各章正文 (chapters[]) interaction: 无（自动逐章生成） rules: - 每章字数目标：1000 字（允许范围 900~1100 字） - 逐章生成，每章生成时需携带前文上下文 - 保持文风统一 - 严格遵循大纲中的事件、角色、时间线 - 对话与叙述合理穿插 ``` **Prompt 模板参考（逐章调用）：** ```text 你是一位优秀的短篇小说作家。 【创作背景】 - 词条：{keywords} - 导语：{selected_synopsis} 【角色表】 {character_table} 【时间线】 {timeline} 【完整大纲】 {full_outline} 【已完成章节】 {previous_chapters_text} （首章时为"无"） 【当前任务】 请撰写 **第{N}章：{chapter_title}** 本章大纲要点： {current_chapter_outline} 要求： 1. 字数控制在 900~1100 字 2. 严格遵循本章大纲的事件与角色安排 3. 与前文保持风格统一、情节连贯 4. 角色姓名、关系、时间线不得出错 5. 合理运用对话、心理描写、环境描写 6. 本章结尾自然衔接下一章（末章除外，末章需有收束感） 请直接输出第{N}章正文： ``` --- ### Step 4：质量检查（Quality Check） ```yaml step_id: quality_check input: 全部章节正文 (chapters[]) + 大纲 + 角色表 + 时间线 output: 质量检查报告 (qa_report) + 修订后正文 (revised_chapters[]) interaction: 无（自动执行，如有问题自动修复） check_items: - 字数检查：每章是否在 900~1100 字范围内 - 时间线检查：是否存在时间线错乱/矛盾 - 角色一致性检查： - 角色名字是否前后一致（无错别字/混用） - 角色关系是否与角色表吻合 - 角色性格/言行是否前后统一 - 逻辑检查： - 因果关系是否合理 - 是否有未交代的情节线 - 是否有前后矛盾的细节 - 叙事质量检查： - 是否存在大段重复 - 节奏是否拖沓或过于仓促 ``` **Prompt 模板参考：** ```text 你是一位严谨的文学编辑 / 质检审校员。 【角色表】 {character_table} 【时间线】 {timeline} 【章节大纲】 {full_outline} 【完整正文】 {all_chapters_text} 请对以上短篇小说进行全面质量检查，逐项审核以下内容： ## 检查清单 ### 1. 字数检查 - 统计每章实际字数 - 标记不在 900~1100 字范围内的章节 ### 2. 时间线一致性 - 检查各章事件的时间顺序是否合理 - 是否存在时间线矛盾 ### 3. 角色一致性 - 角色名字是否全文统一（无笔误、混用） - 角色关系是否与角色表一致 - 角色言行、性格是否前后连贯 ### 4. 逻辑合理性 - 因果关系是否通顺 - 是否有悬而未决的情节线 - 是否有前后矛盾的细节 ### 5. 叙事质量 - 是否有重复段落/表述 - 节奏是否合理 请输出检查报告，格式如下： ## 质量检查报告 | 检查项 | 状态 | 问题描述 | 所在章节 | 修改建议 | |--------|------|----------|----------|----------| 如发现问题，请直接输出修改后的完整正文。 如无问题，请输出："✅ 质量检查通过，无需修改。" ``` --- ### Step 5：通篇复审（Coherence Review） ```yaml step_id: coherence_review input: 质检后正文 (revised_chapters[]) + 原始词条 (keywords[]) + 选定导语 (selected_synopsis) output: 复审报告 (review_report) + 终版正文 (final_chapters[]) interaction: 无（自动执行） review_focus: - 正文是否完整体现所有用户词条 - 正文整体走向是否与导语描述一致 - 主题/基调是否统一 - 结局是否呼应开篇 ``` **Prompt 模板参考：** ```text 你是一位资深的文学复审编辑。 【用户原始词条】 {keywords} 【选定导语】 {selected_synopsis} 【当前正文（质检后版本）】 {revised_full_text} 请进行通篇复审，重点审核： 1. **词条覆盖度**：逐一核对每个词条，是否在正文中得到合理体现？ - 列出每个词条及其在正文中的体现方式/位置 - 标记未体现或体现不足的词条 2. **导语一致性**：正文的核心冲突、角色设定、故事走向是否与导语一致？ - 是否有偏离导语的情节？ 3. **主题统一性**：全文的主题和基调是否前后一致？ 4. **首尾呼应**：结局是否与开篇形成呼应或闭环？ 请输出复审报告： ## 复审报告 ### 词条覆盖情况 | 词条 | 是否体现 | 体现位置/方式 | 备注 | |------|----------|---------------|------| ### 导语一致性：✅/❌ 说明：... ### 主题统一性：✅/❌ 说明：... ### 首尾呼应：✅/❌ 说明：... 如发现问题，请直接输出修正后的完整终版正文。 如无问题，请输出："✅ 复审通过，正文定稿。" ``` --- ### Step 6：书名生成（Title Generation） ```yaml step_id: generate_titles input: 终版正文 (final_chapters[]) + 词条 (keywords[]) + 导语 (selected_synopsis) output: 5 个候选书名 (titles[5]) interaction: 需要用户选择 ``` **Prompt 模板参考：** ```text 你是一位擅长取名的文学编辑。 【词条】{keywords} 【导语】{selected_synopsis} 【正文概要】（取首尾章 + 高潮章核心段落） 请为这篇短篇小说生成 5 个候选书名。 要求： 1. 书名应精炼、有吸引力，长度建议 2~8 个字 2. 5 个书名风格各异： - 一个直白点题型 - 一个意象隐喻型 - 一个悬念引导型 - 一个情感共鸣型 - 一个诗意文艺型 3. 书名应与正文内容高度契合 输出格式： 1. 《xxx》—— 类型：直白点题 | 释义：... 2. 《xxx》—— 类型：意象隐喻 | 释义：... 3. 《xxx》—— 类型：悬念引导 | 释义：... 4. 《xxx》—— 类型：情感共鸣 | 释义：... 5. 《xxx》—— 类型：诗意文艺 | 释义：... ``` **用户交互：** ```text 请选择您最喜欢的书名（回复编号 1-5）， 或提出您的想法，我将为您调整。 ``` --- ## 四、最终输出格式 ```yaml step_id: assemble_output input: 选定书名 + 词条 + 导语 + 终版正文 output: 完整作品 (final_work) ``` **输出模板：** ```markdown # 《{selected_title}》 **词条标签**：{keyword_1} | {keyword_2} | {keyword_3} | ... --- ## 导语 {selected_synopsis} --- ## 正文 ### 第一章：{title} {chapter_1_content} ### 第二章：{title} {chapter_2_content} ... ### 第N章：{title} {chapter_N_content} --- （全文完） ``` --- ## 五、Skill 配置总览 ```yaml skill: name: short_story_creator version: "1.0" description: "基于词条的短篇小说全流程创作引擎" steps: - id: generate_synopsis name: "导语生成" requires_user_input: true # 用户需选择 1/5 retry_on_fail: true max_retries: 2 - id: generate_outline name: "大纲生成" requires_user_input: optional # 用户可确认或调整 retry_on_fail: true max_retries: 2 - id: write_content name: "正文创作" requires_user_input: false mode: sequential_by_chapter # 逐章生成 context_window: full # 携带全部前文 retry_on_fail: true max_retries: 2 - id: quality_check name: "质量检查" requires_user_input: false auto_fix: true # 发现问题自动修复 check_items: - word_count_per_chapter: { min: 900, max: 1100 } - timeline_consistency: true - character_consistency: true - logic_check: true - narrative_quality: true retry_on_fail: true max_retries: 3 - id: coherence_review name: "通篇复审" requires_user_input: false auto_fix: true review_items: - keyword_coverage: true - synopsis_alignment: true - theme_consistency: true - opening_ending_echo: true retry_on_fail: true max_retries: 2 - id: generate_titles name: "书名生成" requires_user_input: true # 用户需选择 1/5 retry_on_fail: true max_retries: 2 - id: assemble_output name: "组装输出" requires_user_input: false user_interaction_points: - after: generate_synopsis # 选择导语 - after: generate_outline # 确认/调整大纲（可选） - after: generate_titles # 选择书名 global_config: language: "zh-CN" tone: "adaptive" # 根据导语自适应 target_chapter_wordcount: 1000 chapter_wordcount_tolerance: 100 ``` --- ## 六、异常处理与边界情况 | 场景 | 处理策略 | |------|----------| | 用户只给了 1 个词条 | 正常执行，在导语阶段进行更大的发散 | | 用户词条过多（>10） | 提示用户精简至核心词条，或自动分类归纳后执行 | | 质量检查发现字数严重超标/不足 | 自动删减/扩写，最多重试 3 次 | | 质量检查多轮仍不通过 | 标记问题章节，提示用户人工干预 | | 复审发现词条未覆盖 | 自动补充相关情节，重新走质检流程 | | 用户对 5 条导语都不满意 | 支持用户给出修改方向，重新生成 | | 用户对 5 个书名都不满意 | 支持用户自定义书名或给出关键词重新生成 | --- ## 七、流程状态机 ``` [START] │ ▼ [AWAITING_KEYWORDS] ──用户输入词条──▶ [GENERATING_SYNOPSIS] │ ▼ [AWAITING_SYNOPSIS_SELECTION] ◄─── 不满意则重新生成 │ 用户选择 │ ▼ [GENERATING_OUTLINE] │ ▼ [AWAITING_OUTLINE_CONFIRM] (可选) │ ▼ [WRITING_CONTENT] ── 逐章循环 │ 全部完成 │ ▼ [QUALITY_CHECKING] │ ┌─ 不通过 ──▶ 自动修复 ──▶ 重新检查（≤3次） │ └─ 通过 │ ▼ [COHERENCE_REVIEWING] │ ┌─ 不通过 ──▶ 自动修复 ──▶ 重回质检 │ └─ 通过 │ ▼ [GENERATING_TITLES] │ ▼ [AWAITING_TITLE_SELECTION] ◄─── 不满意则重新生成 │ 用户选择 │ ▼ [ASSEMBLING_OUTPUT] │ ▼ [END] ``` --- > **文档版本**：v1.0 > **适用场景**：短篇小说（总字数约 3,000 ~ 8,000 字） > **交互节点**：3 个（选导语 → 确认大纲 → 选书名）
