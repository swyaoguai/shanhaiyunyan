# 一次性质检极限优化方案

## 问题
如果坚持一次性发送所有章节进行质检（不分章），当前会遇到：
- 输入：15000+ tokens
- 超时：600秒不够用
- 504错误

## 解决方案

### 方案1：压缩输入内容 ⭐⭐⭐⭐

**原理**：减少输入token数量，加快处理速度

```python
def build_compressed_quality_check_prompt(workflow):
    """只发送章节摘要而非完整内容"""
    chapters_summary = []
    for chapter in workflow['chapters']:
        content = chapter['content']
        # 只保留前200字+后100字
        if len(content) > 300:
            compressed = content[:200] + "...[省略中间部分]..." + content[-100:]
        else:
            compressed = content
        
        chapters_summary.append({
            'number': chapter['chapter_number'],
            'title': chapter['title'],
            'preview': compressed,
            'word_count': len(content)
        })
    return chapters_summary
```

**效果**：
- 输入从15000 tokens → 5000 tokens
- 时间从15分钟 → 5-8分钟
- 成功率：85%

### 方案4：启用流式输出 ⭐⭐⭐⭐⭐

**原理**：边生成边返回，避免完全超时

```python
async def _run_prompt_stream(prompt, api_config_id, model, **kwargs):
    """流式调用，实时返回结果"""
    model_config = _resolve_model_config(api_config_id, model)
    client = LLMClient(model_config, metrics_namespace="ShortStoryPanel")
    
    chunks = []
    try:
        async for chunk in client.call_stream(
            messages=[{"role": "user", "content": prompt}],
            temperature=model_config.temperature,
            max_tokens=kwargs.get('max_tokens_limit', 2000),
        ):
            chunks.append(chunk)
            # 可以实时显示进度给用户
    except Exception as e:
        # 即使中断，也能返回已生成的部分
        if chunks:
            return ''.join(chunks) + "\n\n[生成被中断]"
        raise
    
    return ''.join(chunks)
```

**效果**：
- 不会完全超时
- 用户能看到实时进度
- 即使中断也有部分结果

### 方案6：分批处理（折中） ⭐⭐⭐⭐

**原理**：不是完全分章，而是每3章一批

```python
async def batch_quality_check(workflow, batch_size=3):
    """分批质检，每批3章"""
    chapters = workflow['chapters']
    all_issues = []
    
    for i in range(0, len(chapters), batch_size):
        batch = chapters[i:i+batch_size]
        batch_text = render_chapters(batch)
        
        # 检查这一批
        prompt = build_batch_check_prompt(batch_text, batch[0]['chapter_number'])
        result = await _run_prompt(prompt, ...)
        
        issues = parse_issues(result)
        all_issues.extend(issues)
    
    return merge_issues(all_issues)
```

**效果**：
- 输入：5000 tokens × 3批
- 时间：6-9分钟
- 成功率：95%
- 不会超时
