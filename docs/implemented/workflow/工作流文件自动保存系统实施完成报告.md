### 2. 工作流文件自动保存系统 ✅

**新建文件**: `novel_agent/web/static/app-workflow-auto-save.js`

**实现内容**:
- 创建统一的自动保存处理器映射
- 实现5种文件类型的自动保存处理：
  - `worldbuilding`: 世界观 → 资料库"世界设定"分类
  - `outline`: 大纲 → 项目数据outline
  - `chapter`: 章节 → 协作创作章节列表
  - `characters`: 角色 → 资料库"角色档案"分类
  - `items`: 道具 → 资料库"道具物品"分类

**核心功能**:
- `handleWorkflowAutoSave()`: 主入口函数，处理工作流完成事件
- `autoSaveWorldbuilding()`: 解析世界观JSON并转换为资料库格式
- `autoSaveOutline()`: 解析大纲JSON并转换为章节列表
- `autoSaveChapter()`: 提取章节信息并保存到对应位置
- `autoSaveCharacters()`: 保存角色数据到资料库
- `autoSaveItems()`: 保存道具数据到资料库
- `fetchWorkflowFileContent()`: 获取工作流文件内容
- `convertWorldDataToLibraryItems()`: 世界观数据格式转换
- `convertOutlineToChapters()`: 大纲数据格式转换
- `extractChapterInfo()`: 章节信息提取
- `refreshCurrentModule()`: 刷新当前模块显示

**集成点**: 
- 在 `app-core.js` 的 `sendCopilotMessage()` 函数中集成
- 当收到 `type: 'done'` 事件且包含 `workflow` 数据时自动触发
- 代码位置：第1005-1030行

## 技术特点

### 1. 模块化设计
- 自动保存功能独立封装在单独的JS文件中
- 通过全局函数暴露接口，便于其他模块调用
- 遵循现有代码的模块化架构

### 2. 数据格式转换
- 智能识别不同的数据格式（数组、对象、嵌套结构）
- 自动提取关键信息（章节号、标题、字数等）
- 生成符合前端数据结构的标准格式

### 3. 错误处理
- 每个保存操作都有独立的try-catch
- 失败不影响主流程，只记录错误
- 显示友好的成功/失败提示

### 4. 用户体验优化
- 自动保存后刷新当前模块显示
- 避免重复保存（通过name/title去重）
- 提供清晰的操作反馈（Toast提示）
- 支持手动关闭工作流面板
