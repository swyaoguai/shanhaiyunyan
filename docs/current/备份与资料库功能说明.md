# 备份与资料库功能说明

## 概述

本文档介绍新增的项目备份和资料库管理功能。

## 一、资料库管理

### 1.1 功能说明

资料库用于存储项目相关的参考资料,包括:
- **文档资料** (.txt, .md, .doc, .docx, .pdf等)
- **图片资料** (.jpg, .png, .gif等)
- **参考资料** (.json, .yaml, .xml等)
- **其他文件**

### 1.2 目录结构

```
data/projects/{project_id}/
├── resources/              # 资料库根目录
│   ├── document/          # 文档类资料
│   ├── image/             # 图片类资料
│   ├── reference/         # 参考资料
│   ├── other/             # 其他文件
│   └── metadata.json      # 资料索引文件
```

### 1.3 API接口

#### 上传资料
```http
POST /api/resources/upload
Content-Type: multipart/form-data

参数:
- project_id: 项目ID
- file: 文件
- file_type: 文件类型(可选)
- description: 描述
- tags: 标签(逗号分隔)
```

#### 列出资料
```http
GET /api/resources/list/{project_id}?file_type=document&tags=参考

参数:
- project_id: 项目ID
- file_type: 文件类型过滤(可选)
- tags: 标签过滤(可选)
```

#### 下载资料
```http
GET /api/resources/download/{project_id}/{file_id}
```

#### 更新资料信息
```http
PUT /api/resources/update/{project_id}/{file_id}

参数:
- description: 描述
- tags: 标签
```

#### 删除资料
```http
DELETE /api/resources/delete/{project_id}/{file_id}
```

#### 获取统计信息
```http
GET /api/resources/statistics/{project_id}
```

### 1.4 使用示例

```python
from novel_agent.utils.resource_manager import get_resource_manager

# 获取资料管理器
resource_manager = get_resource_manager()

# 添加资料
resource = resource_manager.add_resource(
    project_id="project_123",
    source_path=Path("参考文档.pdf"),
    file_type="document",
    description="世界观参考资料",
    tags=["世界观", "参考"]
)

# 列出资料
resources = resource_manager.list_resources(
    project_id="project_123",
    file_type="document"
)

# 获取统计
stats = resource_manager.get_statistics("project_123")
```

## 二、项目备份

### 2.1 功能说明

项目备份功能支持:
- **完整备份**: 备份项目的所有内容
- **选择性备份**: 只备份指定的内容
- **备份导出**: 将备份导出为ZIP文件
- **备份导入**: 从ZIP文件导入备份
- **备份恢复**: 从备份恢复项目

### 2.2 备份内容

完整备份包含以下所有数据:

#### 项目目录内的数据
- **世界观设定** (worldbuilding.json) - 小说世界观、力量体系、地理环境等
- **故事大纲** (outline.json) - 故事大纲、章节规划
- **角色设定** (characters.json) - 角色信息、关系网络
- **物品设定** (items.json) - 重要物品、道具设定
- **章节文件** (chapters/) - 所有已创作的章节内容
- **辅助记忆** (aux_memory/) - 辅助记忆分类、条目、资源等
- **资料库** (resources/) - 用户上传的参考资料、文档、图片等
- **前端状态** (client_state/) - 前端界面状态、用户偏好设置
- **模式记忆** (mode_memory/) - 各种写作模式的记忆数据
  - infinite_write/ - 无限续写模式记忆
  - collab_write/ - 协作写作模式记忆

#### 外部数据(data目录)
- **知识库** (knowledge_base/{project_id}/) - 向量数据库、嵌入数据
  - 包含所有量化的记忆文件
  - 章节内容的向量索引
  - 用于语义检索的嵌入数据

### 2.2.1 备份数据详解

**知识库(knowledge_base)说明:**
- 存储位置: `data/knowledge_base/{project_id}/`
- 内容: 使用向量数据库存储的量化记忆
- 作用: 支持语义搜索、智能检索、上下文关联
- 重要性: ⭐⭐⭐⭐⭐ (非常重要,丢失后无法恢复)

**模式记忆(mode_memory)说明:**
- 存储位置: `data/projects/{project_id}/mode_memory/`
- 内容: 各种写作模式的历史记录和状态
- 作用: 保持写作连贯性、记录创作历史
- 重要性: ⭐⭐⭐⭐ (重要,影响写作体验)

**辅助记忆(aux_memory)说明:**
- 存储位置: `data/projects/{project_id}/aux_memory/`
- 内容: 用户偏好、写作风格、约束规则等
- 作用: 个性化写作建议、风格一致性
- 重要性: ⭐⭐⭐⭐ (重要,影响AI输出质量)

### 2.3 备份存储

```
data/backups/
├── backup_20260318_160000.zip
├── backup_20260318_150000.zip
└── ...
```

每个备份ZIP包含:
- `backup_metadata.json` - 备份元数据
- `project/` - 项目目录内的所有文件
  - worldbuilding.json
  - outline.json
  - characters.json
  - items.json
  - chapters/
  - aux_memory/
  - resources/
  - client_state/
  - mode_memory/
- `external/` - 外部数据
  - knowledge_base/ - 知识库向量数据

### 2.3.1 备份大小估算

典型项目的备份大小:
- **小型项目** (10章以内): 5-20 MB
  - 章节文本: 1-5 MB
  - 知识库: 2-10 MB
  - 其他数据: 2-5 MB
  
- **中型项目** (50章左右): 20-100 MB
  - 章节文本: 5-30 MB
  - 知识库: 10-50 MB
  - 资料库: 5-20 MB
  
- **大型项目** (100章以上): 100-500 MB
  - 章节文本: 30-150 MB
  - 知识库: 50-250 MB
  - 资料库: 20-100 MB

### 2.4 API接口

#### 创建备份
```http
POST /api/backup/create

Body:
{
  "project_id": "project_123",
  "backup_type": "full",
  "include_items": ["worldbuilding", "outline", "chapters"]
}
```

#### 列出备份
```http
GET /api/backup/list
```

#### 获取备份信息
```http
GET /api/backup/info/{backup_id}
```

#### 恢复备份
```http
POST /api/backup/restore

Body:
{
  "backup_id": "20260318_160000",
  "target_project_id": "new_project_123",
  "overwrite": false
}
```

#### 下载备份
```http
GET /api/backup/download/{backup_id}
```

#### 导入备份
```http
POST /api/backup/import
Content-Type: multipart/form-data

参数:
- file: 备份ZIP文件
```

#### 删除备份
```http
DELETE /api/backup/delete/{backup_id}
```

### 2.5 使用示例

```python
from novel_agent.utils.backup import get_backup_service

# 获取备份服务
backup_service = get_backup_service()

# 创建完整备份
result = backup_service.create_backup(
    project_id="project_123",
    project_name="我的小说",
    backup_type="full"
)

# 创建选择性备份
result = backup_service.create_backup(
    project_id="project_123",
    project_name="我的小说",
    backup_type="partial",
    include_items=["worldbuilding", "outline", "chapters"]
)

# 列出所有备份
backups = backup_service.list_backups()

# 恢复备份
result = backup_service.restore_backup(
    backup_id="20260318_160000",
    target_project_id="new_project_123",
    overwrite=False
)

# 导出备份
backup_service.export_backup(
    backup_id="20260318_160000",
    export_path=Path("我的备份.zip")
)

# 导入备份
backup_id = backup_service.import_backup(
    backup_file=Path("外部备份.zip")
)
```

## 三、最佳实践

### 3.1 备份策略

1. **定期备份**: 建议每完成重要章节后创建备份
2. **多版本保留**: 保留多个备份版本,以便回退
3. **导出重要备份**: 将重要备份导出到外部存储

### 3.2 资料库管理

1. **分类存储**: 使用文件类型和标签进行分类
2. **添加描述**: 为每个资料添加清晰的描述
3. **定期清理**: 删除不再需要的资料文件

### 3.3 恢复注意事项

1. **检查备份信息**: 恢复前先查看备份详情
2. **使用新项目ID**: 恢复到新项目避免覆盖现有数据
3. **验证恢复结果**: 恢复后检查文件完整性

## 四、故障排除

### 4.1 备份失败

**问题**: 创建备份时失败
**解决方案**:
- 检查磁盘空间是否充足
- 确认项目目录存在且可访问
- 查看日志文件获取详细错误信息

### 4.2 恢复失败

**问题**: 恢复备份时失败
**解决方案**:
- 确认备份文件完整且未损坏
- 检查目标项目ID是否已存在
- 使用 `overwrite=True` 覆盖现有项目

### 4.3 资料上传失败

**问题**: 上传资料文件失败
**解决方案**:
- 检查文件大小是否超过限制
- 确认文件格式是否支持
- 检查项目目录权限

## 五、技术细节

### 5.1 备份格式

备份使用ZIP压缩格式,包含:
- 元数据文件 (JSON格式)
- 项目文件 (保持原始目录结构)

### 5.2 资料索引

资料库使用 `metadata.json` 存储文件索引:
```json
{
  "res_abc123": {
    "id": "res_abc123",
    "project_id": "project_123",
    "filename": "res_abc123.pdf",
    "original_filename": "参考文档.pdf",
    "file_type": "document",
    "mime_type": "application/pdf",
    "file_size": 1024000,
    "file_path": "document/res_abc123.pdf",
    "description": "世界观参考资料",
    "tags": ["世界观", "参考"],
    "created_at": "2026-03-18T16:00:00",
    "updated_at": "2026-03-18T16:00:00"
  }
}
```

### 5.3 安全性

- 所有文件操作使用原子写入
- 备份文件包含完整性校验
- 支持备份加密(未来版本)
