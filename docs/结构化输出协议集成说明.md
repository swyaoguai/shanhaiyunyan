# 结构化输出协议集成说明

## 概述

本文档说明如何将 [`结构化.txt`](../结构化.txt) 中的协议改造成**分层输出协议体系**，并接入协作模式与其他模式。

## 集成方案

已从“单一协议直接套所有模式”调整为“按模式类型分层治理”：

| 模式类型 | 代表模式 | 协议形态 |
| :--- | :--- | :--- |
| 对话型 | `Communicator` | 按场景启用的 Markdown 结构化协议 |
| JSON型 | `Worldbuilder` / `Outliner` / `Evaluator` | 严格 JSON 结构化协议 |
| 正文型 | `ChapterWriter` / `ContinuousWriter` / `Polisher` | 正文型输出豁免协议 |

### 2. 工具模块开发

**文件：[`novel_agent/agents/structured_output.py`](../novel_agent/agents/structured_output.py)**

新增两个工具类：

#### `StructuredOutputValidator`
验证输出是否符合协议：
- `check_linear_flow()` - 检测线性流叙事
- `check_noise()` - 检测口语化噪音
- `check_structure()` - 检测结构化元素
- `validate()` - 完整验证并给出建议

#### `StructuredOutputHelper`
辅助生成结构化输出：
- `format_comparison()` - 生成对比表格
- `format_steps()` - 生成有序步骤列表
- `format_explanation()` - 生成解释列表
- `add_conclusion_prefix()` - 添加核心结论
- `add_warning_block()` - 添加警告引用块

## 协议核心要求

### 对话型协议
- ✅ 总结/确认/比较场景使用 Markdown 结构化表达
- ✅ 轻量追问场景允许自然对话，不强制大段分块
- ✅ 保留降噪、结论前置、表格/列表优先等规则

### JSON 型协议
- ✅ 只输出合法 JSON
- ✅ 禁止 Markdown 包裹
- ✅ 关键字段机读稳定、建议具体可执行

### 正文型豁免协议
- ✅ 只输出最终正文
- ✅ 禁止进度、说明、表格、清单、元信息
- ✅ 保持叙事自然流动

## 使用示例

### Agent提示词中自动应用

- 对话型模式会在总结/确认场景自动切换结构化表达
- JSON 型模式会强制只输出合法 JSON
- 正文型模式会强制只输出正文，不输出结构化报告

### 手动验证输出

```python
from novel_agent.agents.structured_output import StructuredOutputValidator

# 验证Agent输出
result = StructuredOutputValidator.validate(agent_output)

if not result["is_valid"]:
    print("违规项：")
    print(result["linear_flow_violations"])
    print(result["noise_violations"])
    print("\n改进建议：")
    for suggestion in result["suggestions"]:
        print(f"- {suggestion}")
```

### 使用辅助工具生成结构化内容

```python
from novel_agent.agents.structured_output import StructuredOutputHelper

# 生成对比表格
items = [
    {"name": "方案A", "性能": "高", "成本": "低"},
    {"name": "方案B", "性能": "中", "成本": "中"}
]
table = StructuredOutputHelper.format_comparison(items, ["性能", "成本"])

# 生成步骤列表
steps = ["准备环境：安装工具", "配置参数：设置系统"]
steps_text = StructuredOutputHelper.format_steps(steps)

# 添加核心结论
section = StructuredOutputHelper.add_conclusion_prefix(
    "方案分析",
    "方案A更适合当前场景"
)
```

## 链路一致性修复

### PromptManager 与运行时统一
已修复 [`PromptManager.get_system_prompt()`](../novel_agent/prompts/prompt_manager.py:418) 的来源优先级：

1. 自定义配置
2. `prompts/*.md` 文件提示词
3. 代码内置 fallback

这与 [`BaseAgent._load_system_prompt()`](../novel_agent/agents/base_agent.py:352) 的真实运行优先级保持一致，避免了“文件提示词与代码默认提示词双轨漂移”。

## 注意事项

1. **正文型模式豁免**：[`ChapterWriter`](../novel_agent/prompts/chapter_writer.md)、[`ContinuousWriter`](../novel_agent/prompts/continuous_writer.md)、[`Polisher`](../novel_agent/prompts/polisher.md) 不强制结构化 Markdown 输出
2. **JSON 型模式单独治理**：[`Worldbuilder`](../novel_agent/prompts/worldbuilder.md)、[`Outliner`](../novel_agent/prompts/outliner.md)、[`Evaluator`](../novel_agent/prompts/evaluator.md) 采用 JSON 结构化约束，而不是 Markdown 协议
3. **对话型模式按场景启用**：[`Communicator`](../novel_agent/prompts/communicator.md) 只在总结、确认、比较、计划说明时强制结构化
4. **持续监控**：定期检查 Agent 输出质量，根据实际效果迭代协议
