# 小说封面生成功能设计方案

## 1. 目标

新增“小说封面生成”工作台，让用户基于当前项目内容、封面示例模板和自定义元素生成中文网文封面。

第一版只做本地应用内的封面生成、提示词预览、历史保存和导出，不接入公开素材市场或外部发布平台。

核心原则：

- 字体与标题区域提示词由模板固定维护，保证封面标题稳定出现在画面下方三分之一处。
- 人物、背景、道具、氛围等画面元素作为变量，由项目内容自动提取，也允许用户手动补充或覆盖。世界观和时代背景不作为封面必填核心，只用于辅助生成更准确的背景描述。
- 生成前先产出一份“封面描写提示词”，用户确认后再调用图像模型生成封面。

## 2. 兼容接口范围

后端统一封装为 `CoverImageService`，按配置能力自动选择调用方式：

| 能力 | 兼容端点 | 用途 |
|------|----------|------|
| 文生图 | `POST /v1/images/generations` | 根据最终封面提示词直接生成封面 |
| 多模态 Responses | `POST /v1/responses` | 兼容支持 Responses 的图像模型或中转站 |
| Chat Completions 兼容 | `POST /v1/chat/completions` | 兼容把图像生成封装在聊天接口中的中转站 |

配置项建议：

- `api_config_id`：复用现有 API 配置。
- `model`：允许独立选择图像模型。
- `endpoint_mode`：`auto | images_generations | responses | chat_completions`。
- `size`：默认 `1024x1536`，预留 `1024x1024`、`1536x1024`。
- `quality`：`standard | high`，不支持时静默降级并提示。
- `n`：第一版固定 1，后续支持批量 2-4 张。

## 3. 前端交互

入口放在创作侧栏或项目资源区，名称为“封面生成”。首屏直接进入可操作工作台，不做说明页。

页面区域：

- 封面模板：用示例图卡片选择，例如“恐怖诡异”“热血冒险”“红月悬疑”“玄幻群像”“都市霓虹”“古风权谋”。
- 基础文字：书名默认读取当前项目名，作者默认读取用户输入或项目作者名。
- 元素来源：`自动提取项目元素`、`只使用自定义元素`、`项目元素 + 自定义补充`。
- 项目元素预览：展示系统提取出的角色/人物、场景背景、关键道具、氛围色调，并把时代背景/世界观作为背景描述参考。
- 自定义元素：用户可补充或覆盖角色/人物、背景场景、关键道具、画面情绪、色彩风格；时代背景/世界观可选填写，只用于辅助背景描写。
- 提示词预览：点击“生成描写提示词”后展示最终 prompt，允许用户微调。
- 操作按钮：生成封面、重新生成、保存到项目资源、复制提示词、删除历史。

封面示例图第一版可使用内置静态示例，不需要联网获取。

## 4. 提示词生成流程

提示词分两步生成，避免前端散落复杂拼接逻辑。

### 4.1 从项目提取元素

`CoverElementPromptBuilder` 从当前项目资料中提取可用于封面的元素：

- 项目元信息：标题、作者、类型、简介、卖点。
- 背景参考资料：时代背景、世界观、地理环境、力量体系、社会结构、核心冲突，用于辅助场景背景描写。
- 角色资料：主角、关键配角、反派、外貌特征、标志性动作或物件。
- 大纲和章节摘要：高频场景、关键事件、标志性符号。
- 物品和设定：武器、道具、组织标志、特殊能量、异象。

输出结构建议：

```json
{
  "title": "从红月开始",
  "author": "XXXX",
  "genre": "悬疑 / 末世 / 异常事件",
  "world_or_era": "红月笼罩后的近未来城市，夜色压抑，异常事件频发",
  "characters": "沉默的主角背影，站在城市边缘，轮廓孤独",
  "scene_background": "巨大红月、夜空、城市剪影、稀疏星光",
  "symbols_props": "红月、影子、破碎街灯",
  "atmosphere_color": "暗色基调，红黑对比，冷暖光交织",
  "composition": "竖版封面，中心视觉符号突出，下方三分之一保留标题区域",
  "custom_overrides": {}
}
```

用户自定义元素的优先级：

1. 用户明确填写的元素优先。
2. 用户未填写的字段由项目提取补齐。
3. 模板默认元素只作为兜底，不覆盖项目和用户内容。

### 4.2 合成最终封面提示词

最终 prompt 由四段组成：

```text
文字与字体提示词
+ 项目/用户元素描写
+ 构图与画面质量要求
+ 禁止项
```

固定禁止项：

```text
不要生成真实出版社、平台 Logo、水印、二维码；不要出现错别字、乱码文字、多余署名；避免低清晰度、过度模糊、脸部崩坏、手部畸形。
```

## 5. 固定字体模板与元素变量

以下模板保留字体和文字位置提示词，人物、背景、道具、氛围等内容作为主要变量填入；时代背景和世界观只作为背景描述辅助变量。

### 5.1 恐怖诡异模板

固定字体提示词：

```text
文字：“{title}”，小字：“{author}·著”，文字在画面下方三分之一的位置（固定位置），张扬的的字体设计，笔画以硬朗的直线和几何图形构成，金色渐变，文字加粗，紧凑，居中，部分笔画带有光效，文字周围带有光晕。
```

元素变量：

```text
{character_or_subject}，{action_or_power_effect}，{world_or_background}，{color_and_lighting}，画面下方从底部开始向上三分之二处黑色渐变，二维动漫，高清，细节刻画。
```

示例变量填充：

```text
character_or_subject = 发丝凌乱带红光、指甲鲜红锐利、眼部闪黄芒的暗黑男性
action_or_power_effect = 手部环绕红色诡异能量环
world_or_background = 背景暗黑玄幻
color_and_lighting = 红黑光效交织，红黑基调搭配光效明暗对比
```

### 5.2 热血冒险模板

固定字体提示词：

```text
文字：“{title}”，小字：“{author}·著”，文字在画面下方三分之一处位置（固定位置），创意科幻游戏字体设计，描边效果，夸张的笔画弧度，字体造型充满动感且棱角分明，融合尖锐边缘与流畅曲线，营造出大胆、视觉冲击力强的装饰性字体效果，部分笔画进行连接和共用笔画，笔画转角圆角处理，笔画延伸，金色渐变，文字加粗加大，大气磅礴，紧凑，居中，文字带有光效，文字周围有光晕。
```

元素变量：

```text
{character_or_subject}，{action_pose}，{scene_mood}，造型动感，姿势抓人眼球，动感动态感强，梦幻，二维动漫，高清，细节刻画。
```

示例变量填充：

```text
character_or_subject = 海上冒险题材的剑士主角或战斗型角色
action_pose = 战斗姿态，特写
scene_mood = 热血、冒险、强烈视觉冲击
```

说明：用户可以在自定义元素中填写具体作品或角色名；内置模板默认使用泛化角色描述，避免把特定 IP 人物硬编码为系统预设。

### 5.3 红月悬疑模板

固定字体提示词：

```text
文字：“{title}”，小字：“{author}·著”，文字在画面下方三分之一的位置（固定位置），时尚抽象硬朗字体，将文字笔画简化为流畅的几何形状，文字加粗，变形延长，金色渐变，笔画加粗，紧凑，居中，部分笔画带有光效，文字周围带有光晕。
```

元素变量：

```text
{central_symbol_or_scene}，{foreground_character_or_subject}，{world_or_background}，{color_and_lighting}，二维插画，高清，细节刻画。
```

示例变量填充：

```text
central_symbol_or_scene = 画面中心一个巨大的红色月亮，在夜空中显得格外突出，占据画面的大部分空间
foreground_character_or_subject = 月亮下方站着一个剪影背影小人
world_or_background = 点缀着繁星，天空和星球相呼应，增强整体的和谐感
color_and_lighting = 整个画面色调偏暗，以红色为主
```

## 6. 后端设计

建议新增文件：

- `novel_agent/cover_image_service.py`
- `novel_agent/cover_prompt_builder.py`
- `novel_agent/web/routes/cover_images.py`
- `novel_agent/web/models/cover_image.py`
- `novel_agent/tests/test_cover_prompt_builder.py`
- `novel_agent/tests/test_cover_image_service.py`
- `frontend-tests/cover-images.dom.test.js`

建议新增 API：

- `GET /api/cover-images/templates`：返回模板卡片、示例图、字体模板 ID 和可填变量。
- `POST /api/cover-images/prompt-draft`：根据当前项目和用户自定义元素生成封面描写提示词。
- `POST /api/cover-images/generate`：根据确认后的最终 prompt 调用图像模型生成封面。
- `GET /api/cover-images/history`：读取当前项目封面历史。
- `DELETE /api/cover-images/history/{cover_id}`：删除指定封面记录。

核心流程：

```text
前端选择模板和元素来源
  -> /api/cover-images/prompt-draft
  -> ProjectCoverContextExtractor
  -> CoverElementPromptBuilder
  -> CoverPromptBuilder
  -> 返回 element_prompt / final_prompt
  -> 用户确认并点击生成
  -> /api/cover-images/generate
  -> CoverImageService
  -> EndpointAdapter(auto)
  -> 保存图片与 metadata
  -> 返回 image_url / image_path / provider_trace
```

保存路径：

```text
data/projects/<project_id>/covers/
  ├── cover-<timestamp>.png
  └── cover-<timestamp>.json
```

metadata 保存：

- `cover_id`
- `template_id`
- `typography_prompt`
- `element_prompt`
- `final_prompt`
- `source_mode`：`project | custom | project_plus_custom`
- `extracted_context_refs`
- `custom_elements`
- `api_config_id`
- `model`
- `endpoint_mode`
- `size`
- `quality`
- `seed` 或 provider request id
- `provider_trace`

## 7. 错误处理

需要把“配置测试成功但生成失败”的原因说清楚：

- 配置可达，但模型不支持图像生成。
- `/images/generations` 不支持，已尝试 `/responses` 或 `/chat/completions`。
- 中转站返回 404/405，提示切换端点模式。
- 返回的是 URL 时下载失败，保留原始 URL 并提示。
- 返回 base64 时解码失败，展示 provider 原始错误摘要。
- 项目资料不足时，提示用户补充自定义元素，而不是直接生成空泛封面。

错误响应建议包含：

```json
{
  "success": false,
  "error": "当前模型不支持 /v1/images/generations",
  "endpoint_tried": ["/v1/images/generations", "/v1/responses"],
  "api_config_id": "cfg-id",
  "model": "image-model",
  "suggestion": "请切换为支持图像生成的模型，或将端点模式改为 Responses。"
}
```

## 8. 实施阶段

### P1：设计与配置

- 更新方案文档，明确第一版封面生成边界。
- 明确接口模式、模板变量、项目元素提取和保存路径。
- 不改现有创作主链。

### P2：提示词生成闭环

- 实现 `ProjectCoverContextExtractor`。
- 实现 `CoverElementPromptBuilder`。
- 实现 `CoverPromptBuilder`。
- 支持项目元素自动提取和用户自定义元素合并。
- 增加提示词生成单元测试。

### P3：后端生成闭环

- 实现 `/api/cover-images/templates`。
- 实现 `/api/cover-images/prompt-draft`。
- 实现 `/api/cover-images/generate`。
- 支持 `images_generations`、`responses`、`chat_completions`。
- 图片保存到项目 `covers` 目录。

### P4：前端工作台

- 新增封面生成页面。
- 内置示例模板卡片。
- 支持项目元素预览、自定义元素、提示词预览、生成、保存和删除历史。

### P5：兼容增强

- 增加端点自动探测与失败回退。
- 增加 provider trace 展示。
- 增加提示词复制和历史复用。

## 9. 验收标准

- 用户能选择示例模板并生成一张封面。
- 用户能先生成描写提示词，再基于该提示词生成封面。
- 系统能从现有项目中提取角色/人物、场景和关键符号，并把时代背景/世界观作为背景描述辅助信息。
- 用户能自定义相关元素，并覆盖或补充项目自动提取内容。
- 字体提示词固定拼入最终 prompt，标题位置保持在画面下方三分之一处。
- 后端记录真实使用的 `api_config_id`、模型和端点。
- 生成失败时能明确说明是模型、端点、鉴权、下载失败还是项目元素不足。
- 生成结果能保存到当前项目并在重启后展示。
- 不影响现有小说创作、无限续写和 API 设置。
