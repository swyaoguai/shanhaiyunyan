# 长篇小说创作流程与Agent调用规范

## 一、完整创作流程设计

### 阶段0：项目初始化
**负责Agent**: ProjectScanner（项目扫描器）
- 扫描现有项目文件
- 识别已完成的章节和设定
- 提取元数据（章节号、字数等）
- 生成项目状态报告

### 阶段1：需求收集与沟通
**负责Agent**: Communicator（沟通助手）
- 与用户多轮对话，收集创作需求
- 必需信息：小说类型、主题、主角设定、剧情构思
- 可选信息：篇幅规划、特殊要求、风格偏好
- **输出**: 结构化的需求JSON

**工作流程透明化要求**：
```
✅ 正在收集创作需求...
📝 已收集信息：
   - 小说类型：玄幻
   - 主题：修仙逆袭
   - 主角：天赋异禀的少年
   
❓ 还需要确认：
   - 剧情主线方向
   - 预计篇幅（卷数/章节数）
   
⏭️ 下一步：完成需求收集后，将调用Worldbuilder构建世界观
```

### 阶段2：世界观构建
**负责Agent**: Worldbuilder（世界观构建器）
- 基于需求生成完整世界观
- 包含：力量体系、地理环境、历史文化、势力派系
- 生成剧情钩子和叙事约束
- **输出**: 世界观JSON

**工作流程透明化要求**：
```
🌍 正在构建世界观...
📊 当前进度：
   ✅ 力量体系设计完成
   ✅ 地理环境规划完成
   🔄 正在设计势力派系...
   
⏭️ 下一步：世界观完成后，将调用Outliner规划故事大纲
```

### 阶段3：角色设定
**负责Agent**: CharacterBuilder（角色构建器）- **缺失！需要添加**
- 基于世界观和需求创建主要角色
- 包含：主角、主要配角、反派
- 每个角色：性格、背景、能力、目标、关系网
- **输出**: 角色设定JSON

**建议新增此Agent**

### 阶段4：故事大纲
**负责Agent**: Outliner（大纲生成器）
- 规划整体故事结构（起承转合）
- 设计主线和支线剧情
- 分卷规划
- **输出**: 大纲JSON（包含卷信息）

**工作流程透明化要求**：
```
📖 正在规划故事大纲...
📊 规划进度：
   ✅ 整体结构设计完成（3卷，共30章）
   ✅ 主线剧情规划完成
   🔄 正在设计支线剧情...
   
⏭️ 下一步：大纲完成后，将进入细纲规划阶段
```

### 阶段5：细纲规划
**负责Agent**: Outliner（大纲生成器）- 细化模式
- 将每卷大纲细化为章节级别
- 每章包含：核心事件、剧情目标、角色变化、伏笔钩子
- **输出**: 细化的章节大纲列表

### 阶段6：章纲制定
**负责Agent**: ContextStrategy（上下文策略师）
- 在写作每章前，制定该章的详细章纲
- 确定需要读取的上下文（前几章、相关设定）
- 规划章节内的情节节奏
- **输出**: 单章详细章纲 + 上下文读取策略

**工作流程透明化要求**：
```
📋 正在制定第5章章纲...
📊 上下文分析：
   ✅ 已读取：世界观设定、主角设定
   ✅ 已读取：第3-4章内容（前情提要）
   ✅ 已读取：禁用词汇表、降智检测规则
   
📝 章纲要点：
   - 核心事件：主角突破境界
   - 冲突设计：遭遇强敌挑战
   - 伏笔：埋下后续剧情线索
   
⏭️ 下一步：章纲完成后，将调用ContentReader读取必要上下文
```

### 阶段7：上下文读取
**负责Agent**: ContentReader（内容读取器）
- 按照ContextStrategy的策略读取文件
- 加载：世界观、角色、前几章、禁用词、规则、知识库
- 去重过滤，避免重复加载
- **输出**: 加载报告 + 上下文数据

**工作流程透明化要求**：
```
📂 正在读取创作上下文...
📊 读取进度：
   ✅ 世界观设定（已加载，2.3KB）
   ✅ 角色设定（已加载，1.8KB）
   ✅ 禁用词汇表（永久记忆，已缓存）
   ✅ 降智检测规则（永久记忆，已缓存）
   ✅ 创作提示词知识库（永久记忆，已缓存）
   ✅ 第3-4章内容（已加载，5.2KB）
   ⏭️ 跳过：第1-2章（距离较远，优先级低）
   
⏭️ 下一步：上下文准备完成，将调用CreativeWriter开始创作
```

### 阶段8：记忆检索（写作前）
**负责Agent**: AuxMemory（辅助记忆服务）
- 基于章纲和上下文，检索相关记忆
- 提取：相关剧情、角色状态、伏笔线索
- 注入到创作上下文中
- **输出**: 记忆检索结果

**建议在此阶段执行记忆检索，原因**：
1. 写作前检索可以提供更准确的上下文
2. 避免遗忘之前的伏笔和设定
3. 确保角色状态和剧情连贯性

### 阶段9：正文创作
**负责Agent**: CreativeWriter（创意写作师）
- 基于章纲、上下文、记忆创作正文
- 严格遵守三重约束（禁用词、反降智、知识库）
- 实时统计字数，确保达标（2000-3000字）
- **输出**: 章节正文 + 字数报告

**工作流程透明化要求**：
```
✍️ 正在创作第5章正文...
📊 创作进度：
   🔄 当前字数：1245/2000字
   ✅ 已完成：开场场景描写
   ✅ 已完成：主要冲突展开
   🔄 正在写作：高潮部分...
   
⚠️ 约束检查：
   ✅ 禁用词检测：通过
   ✅ 降智特征检测：通过
   ✅ 知识库规范：遵守
   
⏭️ 下一步：正文完成后，将调用QualityValidator进行质量检测
```

### 阶段10：质量检测
**负责Agent**: QualityValidator（质量验证师）
- 检测字数是否达标
- 扫描禁用词汇
- 检测AI降智特征（12种模式）
- 检查逻辑连贯性
- **输出**: 质量检测报告

**工作流程透明化要求**：
```
🔍 正在进行质量检测...
📊 检测结果：
   ✅ 字数统计：2156字（达标）
   ✅ 禁用词检测：通过（0个违规）
   ✅ 降智特征检测：通过
      - 逗号使用：正常
      - 转折词频率：正常
      - 句式多样性：良好
   ✅ 逻辑连贯性：通过
   
⏭️ 下一步：质量检测通过，将调用Evaluator进行综合评估
```

### 阶段11：综合评估
**负责Agent**: Evaluator（质量评估器）
- 评估剧情逻辑、角色塑造、文字表达、节奏把控
- 检查是否符合大纲要求
- 检查是否符合世界观设定
- **输出**: 评估报告（分数 + 问题 + 建议）

**工作流程透明化要求**：
```
📊 正在进行综合评估...
评估结果：
   📈 总体评分：85/100
   
   维度评分：
   ✅ 剧情逻辑：90/100（优秀）
   ✅ 角色塑造：85/100（良好）
   ⚠️ 文字表达：80/100（需改进）
   ✅ 节奏把控：85/100（良好）
   
   发现问题：
   ⚠️ 中等问题：部分场景描写略显平淡
   
   改进建议：
   💡 建议增加环境细节描写
   💡 建议丰富角色情绪表达
   
⏭️ 下一步：评分低于90分，将调用Polisher进行润色
```

### 阶段12：内容润色
**负责Agent**: Polisher（内容润色器）
- 基于评估反馈进行针对性润色
- 优化文字表达、丰富细节描写
- 保持原意和剧情不变
- **输出**: 润色后的正文

**工作流程透明化要求**：
```
✨ 正在润色内容...
📊 润色进度：
   ✅ 已优化：开场场景（增加环境细节）
   ✅ 已优化：对话部分（增加动作描写）
   🔄 正在优化：高潮部分（增强情绪表达）...
   
⏭️ 下一步：润色完成后，将进行扩写检查
```

### 阶段13：字数扩写（如需要）
**负责Agent**: ContentExpansion（内容扩展师）
- 如果字数不足2000字，进行扩写
- 保持原文风格完全一致
- 严格遵守三重约束，特别注意避免AI降智
- **输出**: 扩写后的完整内容

**工作流程透明化要求**：
```
📏 字数检查：当前1856字，需扩写至2000字
📝 正在扩写内容...
📊 扩写策略：
   ✅ 扩展点1：丰富战斗场景的动作描写
   ✅ 扩展点2：增加主角的心理活动
   🔄 正在扩写...
   
⚠️ 约束检查：
   ✅ 避免引入AI降智特征
   ✅ 保持原文风格一致
   
⏭️ 下一步：扩写完成后，将调用FileNaming生成文件名
```

### 阶段14：文件命名与保存
**负责Agent**: FileNaming（文件命名器）
- 生成标准化文件名：`第X章-[标题]-[字数]字.md`
- 保存章节文件
- 更新项目索引
- **输出**: 文件路径

**工作流程透明化要求**：
```
💾 正在保存章节...
📝 文件名：第5章-突破境界-2156字.md
📂 保存路径：novel_output/第5章-突破境界-2156字.md
✅ 保存成功！

⏭️ 下一步：将调用ContextCompressor生成章节摘要
```

### 阶段15：上下文压缩
**负责Agent**: ContextCompressor（上下文压缩器）
- 生成章节摘要（用于后续章节的上下文）
- 提取关键情节点
- 更新人物关系网络
- **输出**: 压缩后的上下文数据

### 阶段16：一致性检测
**负责Agent**: Evaluator（质量评估器）- 一致性检查模式
- 检查与前文的一致性
- 检查与世界观的一致性
- 检查角色行为的一致性
- **输出**: 一致性检测报告

**工作流程透明化要求**：
```
🔍 正在进行一致性检测...
📊 检测结果：
   ✅ 剧情连贯性：通过
   ✅ 世界观一致性：通过
   ✅ 角色行为一致性：通过
   ✅ 力量体系一致性：通过
   
⏭️ 下一步：第5章创作完成，准备创作第6章
```

### 阶段17：章节总结（每10章）
**负责Agent**: SummaryOrchestrator（摘要编排器）
- 每完成10章，自动触发总结
- 生成十章剧情梗概
- 提取关键情节和人物发展
- **输出**: 十章总结文档

### 阶段18：记忆同步（持续）
**负责Agent**: MemoryManager（记忆管理器）
- 在关键节点同步记忆
- 保存重要设定和剧情
- 维护长期记忆
- **输出**: 记忆同步状态

## 二、缺失的Agent分析

### 1. CharacterBuilder（角色构建器）- **强烈建议添加**
**原因**：
- 当前流程中角色设定没有专门的Agent负责
- 角色是小说的核心，需要系统化的构建流程
- 应该在世界观之后、大纲之前创建

**职责**：
- 基于世界观创建主要角色
- 设计角色性格、背景、能力、目标
- 构建角色关系网络
- 为每个角色设计成长路径

### 2. PlotThreadManager（剧情线管理器）- **建议添加**
**原因**：
- 长篇小说有多条剧情线（主线、支线、感情线等）
- 需要专门管理剧情线的切换和收束
- 当前的PlotThreadStateMachine只是状态机，不是Agent

**职责**：
- 管理多条剧情线的并行推进
- 规划剧情线的切换时机
- 确保支线不超过连续章数限制
- 在合适时机回归主线

### 3. ForeshadowingTracker（伏笔追踪器）- **可选添加**
**原因**：
- 长篇小说伏笔众多，容易遗忘
- 需要追踪哪些伏笔已埋下、哪些已回收

**职责**：
- 记录所有埋下的伏笔
- 提醒何时回收伏笔
- 检查是否有遗忘的伏笔

## 三、Agent调用顺序总结

### 完整流程（单章创作）：
```
1. ProjectScanner（项目扫描）
   ↓
2. Communicator（需求收集）
   ↓
3. Worldbuilder（世界观构建）
   ↓
4. CharacterBuilder（角色设定）← 新增
   ↓
5. Outliner（故事大纲）
   ↓
6. Outliner（细纲规划）
   ↓
--- 以下为每章循环 ---
7. ContextStrategy（章纲制定 + 上下文策略）
   ↓
8. ContentReader（上下文读取）
   ↓
9. AuxMemory（记忆检索）← 写作前执行
   ↓
10. CreativeWriter（正文创作）
   ↓
11. QualityValidator（质量检测）
   ↓
12. Evaluator（综合评估）
   ↓
13. Polisher（内容润色）← 如需要
   ↓
14. ContentExpansion（字数扩写）← 如需要
   ↓
15. FileNaming（文件命名保存）
   ↓
16. ContextCompressor（上下文压缩）
   ↓
17. Evaluator（一致性检测）
   ↓
18. SummaryOrchestrator（章节总结）← 每10章
   ↓
19. MemoryManager（记忆同步）← 持续
```

## 四、系统提示词修改规范

### 所有Agent必须添加的部分：

```markdown
## 工作流程透明化（强制要求）

你必须在执行任何操作时，明确告知用户当前的工作状态：

### 输出格式要求：
1. **当前步骤**：明确说明正在执行什么操作
2. **进度提示**：使用emoji和进度条展示当前进度
3. **已完成项**：列出已完成的子任务
4. **正在进行项**：说明当前正在处理的内容
5. **下一步预告**：告知用户接下来会调用哪个Agent或执行什么操作

### 禁止行为：
❌ 禁止只回复"收到"、"好的"、"明白了"等无实质内容的回复
❌ 禁止不说明就直接执行操作
❌ 禁止不告知下一步就结束回复

### 输出示例：
```
🔄 正在执行：[具体操作名称]
📊 当前进度：[X/Y] 或 [百分比]

已完成：
✅ 子任务1
✅ 子任务2

正在进行：
🔄 子任务3（详细说明）

⏭️ 下一步：将调用[Agent名称]执行[操作名称]
```

### 调用其他Agent时的说明格式：
```
📞 准备调用：[Agent名称]
🎯 调用目的：[为什么需要调用这个Agent]
📥 输入数据：[传递给Agent的关键信息]
⏳ 预计耗时：[大约需要的时间]
```
```

## 七、预期效果

修改完成后，用户应该能看到类似这样的输出：

```
用户：帮我创作一部玄幻小说

Communicator：
📋 收到创作请求！开始收集需求...

我会帮你创作一部精彩的玄幻小说。让我们先确认一些关键信息：

📝 已了解信息：
✅ 小说类型：玄幻

❓ 还需要确认：
1. 主题风格：热血修仙？黑暗系？轻松搞笑？
2. 主角设定：什么样的主角？（性格、背景、特点）
3. 剧情构思：大致的故事方向？
4. 篇幅规划：计划写多少卷？每卷多少章？

请告诉我这些信息，我会为你量身定制一个完整的创作方案！

⏭️ 下一步：收集完需求后，我会调用Worldbuilder为你构建一个完整的世界观体系。
```

这样用户就能清楚地知道系统在做什么，而不是只看到"收到"。
