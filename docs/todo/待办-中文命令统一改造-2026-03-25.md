# 待办：Copilot 中文命令统一改造

日期：2026-03-25

## 背景

当前 Copilot 存在三条并行命令链路：

1. 英文斜杠显式命令
2. 中文控制短语
3. 中文自然语言意图识别

对真实中文用户不友好，且协议分散，容易出现维护和体验割裂。

## 目标

把用户可见和用户可输入的命令入口统一为中文。

原则：

- 用户界面只展示中文命令
- 用户输入支持纯中文命令
- 英文命令不再要求用户理解或记忆
- 内部仍可保留统一标准命令对象，供路由和执行使用

## 预期方案

### 一、统一命令归一化入口

在后端新增统一命令解析层，优先级高于普通意图识别：

- `开始创作`
- `/开始创作`
- `生成世界观`
- `/生成世界观`
- `生成大纲`
- `/生成大纲`
- `续写章节 3`
- `/续写章节 3`
- `暂停创作`
- `继续创作`
- `查看进度`
- `取消创作`

全部归一化为内部命令对象：

- `create`
- `worldbuild`
- `outline`
- `chapter`
- `pause`
- `resume`
- `status`
- `cancel`

### 二、前端中文化

前端只保留中文展示：

- 快捷命令按钮只显示中文
- 自动补全候选只显示中文主标题
- 英文斜杠协议不再暴露为主文案
- 错误提示、帮助提示、示例文案全部中文

### 三、移除当前割裂逻辑

需要收口的地方：

- 英文斜杠显式命令解析
- 中文控制短语单独分支
- 自然语言意图中的“像命令”的固定短语

目标是把“固定命令类输入”统一走同一解析入口，只把真正自由表达留给 Router。
