# 遗留问题修复方案

> 上次审计共发现 13 个 bug/架构问题，已完成 12 个。本文档记录剩余 1 个问题及其修复方案。

## 背景

P2-2（SessionStore 锁泄漏）原方案是在 `clear_project_cache()` 中清理锁对象，但测试 `test_session_store_delete_and_clear_project_cache_keep_lock_identity` 要求锁在 `delete()` 和 `clear_project_cache()` 后保持同一对象引用（即 identity 不变）。

`ChatSessionStore` 的做法是在 `delete()` 和 `clear_project_cache()` 中均清理锁，两个 Store 设计不一致。

核心矛盾：`SessionStore` 的锁是 session 粒度的 `RLock`/`asyncio.Lock`，按需创建但永不释放。长期运行的服务中，若不断创建新 session，锁字典会无限增长。

---

## 问题 P2-2：SessionStore 锁字典无限增长

### 文件

`novel_agent/agents/session_store.py`

### 现状

```python
# line 202-207
self._locks: Dict[str, threading.RLock] = {}
self._locks_guard = threading.Lock()
self._async_locks: Dict[str, asyncio.Lock] = {}
self._async_locks_guard = threading.Lock()
```

`_get_lock()` 和 `_get_async_lock()` 按 `{project}::{session}` 键创建锁，但没有任何上限或淘汰机制。

### 约束

1. `delete()` 和 `clear_project_cache()` 后，现有测试要求锁 identity 不变（同一 session 重新访问时拿到同一个锁对象）
2. `ChatSessionStore` 在 `delete()` 和 `clear_project_cache()` 中清理锁，两个 Store 行为不一致

### 方案：给锁字典加容量上限 + LRU 淘汰

在 `_SessionStoreBase.__init__` 中增加常量 `_MAX_LOCK_POOL_SIZE = 500`。

在 `_get_lock()` 和 `_get_async_lock()` 创建新锁之前，如果字典大小已达上限，清理最早创建的条目（dict 保持插入顺序）。

这种方式：
- 不破坏现有 `delete()`/`clear_project_cache()` 的行为
- 不需要修改测试
- 锁字典大小被限制在 500 以内
- 常规使用中（<500 并发 session），锁 identity 仍保持稳定

### 具体改动

#### 步骤 1：添加常量

```python
# _SessionStoreBase.__init__ 中，在 self._async_locks_guard 之后
self._MAX_LOCK_POOL_SIZE = 500
```

#### 步骤 2：修改 `_get_lock()`

```python
def _get_lock(self, session_id: str, project_id: str = "") -> threading.RLock:
    key = self._cache_key(session_id, project_id)
    with self._locks_guard:
        if key not in self._locks:
            if len(self._locks) >= self._MAX_LOCK_POOL_SIZE:
                oldest = next(iter(self._locks))
                del self._locks[oldest]
            self._locks[key] = threading.RLock()
        return self._locks[key]
```

#### 步骤 3：修改 `_get_async_lock()`

```python
def _get_async_lock(self, session_id: str, project_id: str = "") -> asyncio.Lock:
    key = self._cache_key(session_id, project_id)
    with self._async_locks_guard:
        lock = self._async_locks.get(key)
        if lock is None:
            if len(self._async_locks) >= self._MAX_LOCK_POOL_SIZE:
                oldest = next(iter(self._async_locks))
                del self._async_locks[oldest]
            lock = asyncio.Lock()
            self._async_locks[key] = lock
        return lock
```

### 验证

1. 现有测试通过：`pytest novel_agent/tests/test_session_id_security.py -v`
2. 锁池上限生效：创建 >500 个不同 session 的锁，验证 `len(store._locks) <= 500`
