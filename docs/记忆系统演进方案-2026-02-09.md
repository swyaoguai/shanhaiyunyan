# 记忆系统演进方案（详细版）

> 日期：2026-02-09  
> 适用项目：小说创作 Agent（当前仓库）  
> 方案定位：**保留现有主记忆链路，增量引入“辅助记忆层”**  
> 参考对象：memU（能力借鉴）、VoltAgent 生态（工程抽象借鉴）

---

## 1. 背景与结论

当前系统已经具备稳定的创作主链路记忆能力（章节续写状态、上下文压缩、项目隔离）。  
评估后确认：

1. **不建议替换现有主记忆系统**（会损失章节状态机能力与连续性保障）。
2. **建议引入辅助记忆层**，重点吸收 memU 的可用能力：
   - 分层记忆模型（Resource / Item / Category）
   - 自动分类与摘要更新
   - 检索双模式（快速检索 + 深推理）
   - 作用域过滤（project_id / user_id）
3. VoltAgent awesome 仓库本身是资源导航集合，不是可直接嵌入的记忆实现；其价值在“接口抽象与工程组织方式”参考。

---

## 2. 目标与非目标

### 2.1 目标

- 在不破坏现有创作工作流前提下，新增“长期偏好/习惯/主题兴趣”记忆能力。
- 支持用户在 Web 端进行可视化编辑（增删改查、启停、回滚）。
- 提升长期对话与创作一致性，同时控制 token 成本。
- 保持项目级隔离，避免跨项目串话与污染。

### 2.2 非目标（本阶段不做）

- 不替换现有 `SessionStore + ContextManager` 主链路。
- 不将辅助记忆作为“唯一事实源”。
- 不引入高侵入式重构（如整体替换 Agent 执行框架）。
- 不强依赖外部云服务才能运行。

---

## 3. 现状盘点（当前系统）

### 3.1 现有主记忆链路（必须保留）

- **会话持久化与续写上下文**  
  `SessionStore` 提供章节、剧情点、角色状态、版本号等核心状态维护。
- **上下文管理与压缩优化**  
  `ContextManager` 提供按角色/任务的上下文隔离、优先级与压缩策略。
- **外部可选记忆服务接口（Wensi）**  
  目前为可选 stub，不影响主流程可用性。

### 3.2 已存在的可视化基础

- Web 端已有“资料库”模块（分类、条目、导入、编辑）。
- 可复用该 UI 模式扩展“辅助记忆中心”。

---

## 4. 差距分析

当前系统强在“章节一致性”，弱在“长期偏好沉淀自动化”：

1. 长期记忆结构偏分散，缺少统一层次模型。
2. 用户偏好抽取多依赖人工维护，自动归纳能力不足。
3. 对复杂问题的记忆检索路径缺少分层决策（先快后深）。
4. 缺少“辅助记忆可解释引用链路”（记忆来源追踪）。

---

## 5. 演进原则（强约束）

1. **主链路优先**：章节事实、剧情状态永远高优先级于辅助记忆。
2. **增量可回滚**：任何新能力以独立模块形式接入，支持一键降级。
3. **双写可观测**：新增阶段先双写日志与指标，确认稳定后再默认启用。
4. **可编辑可审计**：用户可在 UI 中看见、修改、禁用辅助记忆项。
5. **项目隔离**：所有辅助记忆默认绑定 `project_id`，可选 `user_id` 细分。

---

## 6. 目标架构（双层记忆）

```text
[主记忆层：强约束]
SessionStore + ContextManager + ProjectData
  └─ 用途：章节连续性、剧情一致性、流程状态

[辅助记忆层：弱约束]
Aux Memory (Resource/Item/Category)
  └─ 用途：长期偏好、写作风格、主题兴趣、习惯规则
  └─ 注入：低优先级提示片段，不覆盖主事实
```

---

## 7. 从 memU 借鉴的能力清单（按优先级）

## P0（首批上线）

### A. 分层记忆模型（必须）
- Resource：来源（对话片段、文档、笔记、用户输入）
- MemoryItem：抽取后的记忆条目（摘要/类型/标签/权重）
- MemoryCategory：主题分类（偏好、风格、禁忌、关系）

### B. 作用域过滤（必须）
- where 过滤思想：按 `project_id`（必选）+ `user_id`（可选）过滤。

## P1（次阶段）

### C. 自动分类与类别摘要更新
- 新增记忆时自动归类；
- 分类摘要增量更新，便于快速注入。

### D. 双模式检索
- `fast`：向量/关键词快速召回（毫秒级）
- `deep`：LLM 深推理重排（秒级，按需触发）

## P2（可选增强）

### E. 引用链路（ref）
- 生成“该建议来自哪些记忆条目”的追踪关系；
- 增强可解释性与编辑可控性。

---

## 8. 从 VoltAgent awesome 借鉴点（仅工程思路）

1. 接口分层与可替换后端思想（内存/SQL/向量存储切换）。
2. Workflow 可中断/恢复理念（本项目已有能力，可继续强化）。
3. 观察性与调试面板思路（对记忆注入过程可视化）。

> 注意：awesome 仓库本身不是实现库，不直接集成代码。

## 10. API 设计（提案）

## 10.1 基础 CRUD

- `GET /api/aux-memory/categories`
- `POST /api/aux-memory/categories`
- `PATCH /api/aux-memory/categories/{id}`
- `DELETE /api/aux-memory/categories/{id}`

- `GET /api/aux-memory/items`
- `POST /api/aux-memory/items`
- `PATCH /api/aux-memory/items/{id}`
- `DELETE /api/aux-memory/items/{id}`

- `POST /api/aux-memory/resources/import`（导入文本/文件）

## 10.2 检索与注入

- `POST /api/aux-memory/retrieve`
  - 参数：
    - `query`
    - `mode: fast|deep`
    - `where: {project_id, user_id}`
    - `top_k`
- `POST /api/aux-memory/injection-preview`
  - 返回实际注入 prompt 片段（用于调试与解释）

## 10.3 版本与回滚

- `GET /api/aux-memory/history`
- `POST /api/aux-memory/rollback`

## 11. 注入优先级策略（核心）

写作前构建上下文时，按以下顺序拼装：

1. 章节硬状态（SessionStore）
2. 工作流与上下文主信息（ContextManager）
3. 项目结构化资料（角色/世界观/大纲）
4. **辅助记忆摘要（Aux Memory，低优先级）**

冲突规则：

- 同主题冲突时，主层覆盖辅助层；
- 辅助层只给“建议约束”，不直接改写剧情事实；
- 可配置“禁用辅助记忆注入”开关用于问题排查。

---

## 12. 可视化设计（你关心的用户能力）

## 12.1 页面结构

新增“长期记忆中心”页面：

- 左侧：分类树（可启停）
- 中间：记忆条目列表（检索、筛选、批量操作）
- 右侧：详情编辑（summary/details/tags/来源）

## 12.2 用户可做操作

- 新增/编辑/删除分类与条目
- 启用/禁用条目（不删除）
- 查看来源与引用链
- 查看被注入记录（最近 N 次）
- 回滚到历史版本

## 12.3 与现有资料库关系

- 资料库继续作为创作资料编辑区；
- 长期记忆中心作为“行为偏好与长期事实沉淀区”；
- 两者可互相导入，但默认隔离。

---

## 13. 分阶段实施计划（里程碑）

## M1：基础落地（1 周）
- 建立 Aux Memory 数据模型与本地持久化
- 完成分类/条目 CRUD API
- 完成最小 UI（列表+编辑）
- 支持手工注入预览

验收：能稳定 CRUD，且不影响现有创作流程。

## M2：自动化增强（1~2 周）
- 上线自动分类与分类摘要更新
- 上线 `retrieve fast` 与 where 过滤
- 注入策略接入写作链路（默认低优先级）

验收：同等输入下，续写一致性不下降，提示词长度可控。

## M3：深检索与可解释（1 周）
- 上线 `retrieve deep`
- 上线引用链与来源追踪
- 上线历史回滚与调试面板

验收：复杂请求命中率提升，用户可追踪“为什么注入这条记忆”。

---

## 14. 风险与回滚策略

## 风险

1. 自动抽取误差导致错误偏好沉淀。
2. 注入过多引发上下文噪声与风格漂移。
3. 数据增长导致检索延迟波动。

## 规避

- 引入条目 `enabled` 与 `score` 阈值；
- 注入上限（条目数、字数）硬限制；
- 周期归档与冷数据清理。

## 回滚

- 一键关闭辅助注入开关；
- 仅保留主链路（SessionStore + ContextManager）运行；
- 辅助层数据保留，不参与生成。

---

## 15. 验收标准（DoD）

1. 功能：
   - 辅助记忆 CRUD 全通；
   - 检索（fast）可用；
   - 注入可预览可关闭。
2. 质量：
   - 主创作功能回归通过；
   - 无跨项目串话；
   - 关键接口有日志与错误码。
3. 体验：
   - 用户可视化编辑顺畅；
   - 可解释“记忆来源与注入理由”。

---

## 16. 最终决策

- **保留主记忆系统不变**（创作连续性核心资产）。  
- **新增辅助记忆层并可视化编辑**（先人工可控，再自动增强）。  
- **优先吸收 memU 的能力模式，不直接整体引入其运行时依赖**。  
- **参考 VoltAgent 的工程分层思路，不做框架替换。**
