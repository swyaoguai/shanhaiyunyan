## 三、Findings

### 1. 中等：`style.css` 混入了与本轮治理目标无关的编辑器布局重写，且缺少对应回归保护

**位置**

1. `novel_agent/web/static/style.css:421`
2. `novel_agent/web/static/style.css:432`

**现象**

`editor-container` 的行为被整体改写为：

1. `width: 100%`
2. `max-width: none`
3. `margin: 0`

同时删除了原先针对：

1. 左侧栏收起
2. 右侧 Copilot 收起
3. 双侧收起/沉浸模式

的宽度调节逻辑，只保留了 `focus-mode` 下的 padding 调整。

**风险**

这会直接影响整个编辑器区域的阅读宽度和留白策略，不仅限于本次治理涉及的设置页或短篇页。  
当前新增的 Python 文本回归与 DOM 回归都没有覆盖这段改动，因此一旦它造成正文编辑区域过宽、布局跳变或沉浸模式体验下降，现有测试不会报警。

**判断依据**

本轮治理文档关注的是：

1. 设置页拆层
2. 短篇页拆层
3. 样式收口
4. DOM 测试补强

并未要求重写全站编辑器主容器布局。因此这段修改属于范围外改动，风险高于一般样式微调。

---

### 2. 低：前端 DOM 测试入口目前只覆盖了最小成功路径，对移动端样式与弹层交互没有真实保护

**位置**

1. `frontend-tests/settings-short-story.dom.test.js:64`
2. `frontend-tests/settings-short-story.dom.test.js:124`
3. `package.json:5`

**现象**

当前 DOM 测试覆盖了：

1. 设置页非法 URL 渲染
2. Skills 恶意文本转义
3. 短篇页跳转到设置 API 页
4. 切换 API 配置后模型列表同步

但没有覆盖：

1. 设置页弹层 `#config-edit-modal` 的打开/关闭行为
2. 移动端下 `settings` 样式类的实际布局回归
3. `toggle-kb-key` 这类嵌入式按钮的真实点击区
4. 设置页按钮在窄屏下的排列与可操作性

**风险**

这不构成当前阻断问题，但意味着这轮样式收口后的回归保护还偏“结构级”，还没有进入“可视布局级”或“组件交互级”。

**备注**

这条不是要求立刻返工，而是说明“已有 DOM 测试”不等于“覆盖了样式回归风险”。

---

### 3. 低：设置页样式体系已成型，但仍存在少量内联样式作为状态补丁，后续继续扩展时容易回流

**位置**

1. `novel_agent/web/static/settings/app-settings-renderers.js:69`
2. `novel_agent/web/static/settings/app-settings-renderers.js:115`
3. `novel_agent/web/static/settings/app-settings-renderers.js:245`
4. `novel_agent/web/static/settings/app-settings-renderers.js:589`

**现象**

当前仍保留一部分内联样式，主要分三类：

1. 动态渐变背景
2. 条件禁用时的 `opacity / pointer-events`
3. 少量颜色或边距补丁

这些中大部分是合理保留的，但如果后续继续往这个模式上叠加，很容易重新回到“模板里塞状态样式补丁”的方向。

**风险**

目前不是 bug，更像是治理边界提醒：  
如果下一轮继续在 `renderers` 里增加条件性 `style=""`，样式系统会再次失控。

## 六、建议

### 建议优先级 P1

1. 单独确认 `editor-container` 那段范围外布局改动是否为有意变更。
2. 如果不是本轮目标的一部分，应拆出去单独评审，或回到独立任务里处理。

### 建议优先级 P2

1. 为设置页配置编辑弹层补 1 条 DOM 测试。
2. 为移动端关键交互补 1 条最小可视行为测试，哪怕只是 class 级断言也比没有好。

### 建议优先级 P3

1. 后续继续治理 settings/short-story 时，新增模板尽量继续走 class，不要回流到大段 `style=""`。

本轮前端治理的主线质量是合格的，设置页与短篇页的结构化程度、样式可控性和回归保护都比之前明显更好。  
当前最值得关注的不是 settings/short-story 主体代码，而是 `style.css` 中混入的那段全局编辑器布局改动，它超出了本轮治理的直接目标，且没有被新增测试覆盖。
