# Project Documentation Index

This directory keeps project documentation grouped by current usefulness and topic. Root-level files such as `README.md`, `CHANGELOG.md`, `MIGRATION_GUIDE.md`, and `START_TESTING.md` remain in the project root because they are common entry points.

## Current

- `current/` - Current user-facing and API documentation.
- `current/API.md` - REST API reference.
- `current/使用说明.md` - Chinese user guide.
- `current/备份与资料库功能说明.md` - Backup and resource-library feature guide.

## Design

- `design/multi-agent/` - Multi-agent architecture and orchestration proposals.
- `design/workflow/` - Writing workflow, outline, and worldbuilding designs.
- `design/memory/` - Memory-system design notes.
- `design/knowledge-base/` - Knowledge-base and Wiki design notes.
- `design/skills/` - Skill integration and calling design notes.

## Implemented

- `implemented/` - Documents for features that have already been delivered or folded into the codebase.
- `implemented/multi-agent/` - Completed multi-agent workflow/chat execution work.
- `implemented/short-story/` - Completed short-story feature notes.
- `implemented/novel-to-script/` - Completed novel-to-script feature notes.
- `implemented/quality-review/` - Completed quality-review workflow notes.
- `implemented/workflow/` - Completed workflow and structured-output notes.
- `implemented/skills/` - Completed Skill migration and extension notes.

## Reports

- `reports/reviews/` - Codebase reviews and audit reports.
- `reports/progress/` - Implementation progress records.
- `reports/testing/` - Test and fix reports.

## Release And Todo

- `release/` - Release notes and version-specific testing docs.
- `todo/` - Open plans, unresolved items, and future governance notes.

## Reference And Archive

- `reference/` - External or vendor reference material.
- `archive/legacy/` - Old design documents retained for history.
- `archive/misc/` - Temporary/generated files kept out of the project root.
