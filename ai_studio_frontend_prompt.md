# Google AI Studio Build Mode - 山海·云梦 长篇小说创作软件前端提示词

## 提示词（直接复制到AI Studio Build模式中）

---

请为我构建一个名为「山海·云梦」的长篇小说创作软件PC端Web前端界面，使用 React + TypeScript + TailwindCSS + shadcn/ui 技术栈。这是一个专业级的小说写作工具，核心设计理念是"Obsidian式知识库 + AI辅助创作"。

## 设计风格总纲：玻璃拟态（Glassmorphism）浮动式UI

整体视觉风格参考以下设计语言，贯穿所有页面：

### 核心视觉特征
1. **毛玻璃卡片（Glassmorphism Cards）**：所有卡片使用半透明白色背景 + backdrop-blur模糊效果，CSS示例：`background: rgba(255,255,255,0.72); backdrop-filter: blur(16px); border: 1px solid rgba(255,255,255,0.3);`
2. **浮动式侧边导航**：左侧导航栏不贴边，是一个悬浮的圆角胶囊形图标栏，垂直居中于页面左侧，带毛玻璃背景和柔和阴影
3. **超大圆角**：卡片圆角16-20px，按钮和标签为胶囊形（border-radius: 9999px），输入框圆角12px
4. **柔和渐变背景**：页面底层背景使用淡薄荷绿到淡青色的微妙径向渐变（`radial-gradient(ellipse at top left, #E0F7FA 0%, #E8F5E9 30%, #F1F8E9 60%, #FAFAFA 100%)`）
5. **嵌套卡片层级**：卡片内部可再嵌套子卡片，通过透明度和阴影区分层级
6. **漫散柔和阴影**：不使用硬边框分割，用大范围柔和阴影区分层级（`box-shadow: 0 8px 32px rgba(0,0,0,0.06), 0 2px 8px rgba(0,0,0,0.04)`）
7. **胶囊形Tab导航**：顶部功能切换使用浮动的胶囊形标签组，选中态有浅色填充
8. **彩色状态标签**：使用胶囊形Badge标识状态（绿色=正常、蓝色=标记、红色=警告、橙色=进行中）
9. **色彩编码进度条**：进度条颜色根据百分比渐变（红0-30%、橙30-70%、绿70-100%）
10. **图标+数字统计卡**：关键数据展示使用"左侧彩色图标圆 + 右侧数字和标签"布局

### 日间主题色彩
- 页面渐变背景：薄荷绿/淡青渐变
- 卡片背景：`rgba(255,255,255,0.72)` 带 `backdrop-blur(16px)`
- 卡片边框：`rgba(255,255,255,0.3)` 或 `rgba(0,0,0,0.05)`
- 主色调：`#0EA5E9`（天青蓝，用于主按钮和激活态）
- 辅助色：`#10B981`（翡翠绿，用于成功/进度）
- 文字主色：`#1E293B`
- 文字次色：`#64748B`
- 文字辅助：`#94A3B8`

### 夜间主题色彩
- 页面背景：深灰蓝渐变（`#0F172A` 到 `#1E293B`）
- 卡片背景：`rgba(30,41,59,0.8)` 带 `backdrop-blur(16px)`
- 卡片边框：`rgba(255,255,255,0.08)`
- 主色调：`#38BDF8`（亮天青蓝）
- 辅助色：`#34D399`（亮翡翠绿）
- 文字主色：`#F1F5F9`
- 文字次色：`#94A3B8`
- 文字辅助：`#64748B`

### 动效与交互
- 所有状态切换：`transition-all duration-300 ease-out`
- 卡片hover：微微上浮2px + 阴影加深
- 按钮hover：亮度提升 + 微缩放`scale(1.02)`
- 悬浮窗出现/消失：`opacity + transform scale` 组合动画
- 页面切换：`fade + slideUp` 组合过渡

## 三、AI悬浮助手（核心功能）

### 3.1 悬浮窗设计
- **默认状态**：圆形悬浮按钮（56px），毛玻璃背景+天青蓝渐变边框，带呼吸灯脉冲动画
- **点击展开**：展开为毛玻璃聊天面板（380px宽，500px高，可拖拽调整）
- **位置记忆**：localStorage存储上次位置，下次打开恢复
- **自由拖拽**：可拖到屏幕任意位置
- **边框吸附**：靠近边缘自动吸附，吸附后变为侧边抽屉
- **最小化**：双击标题栏回到悬浮按钮

### 3.2 助手聊天界面
- **顶部栏**（毛玻璃）：
  - 当前角色头像+名称（点击切换角色）
  - 模型选择下拉框
  - 新对话 | 历史记录 | 最小化/关闭
- **角色切换面板**（展开为卡片网格）：
  - 预设角色：
    - 正文写作专家
    - 大纲规划大师
    - 世界观构建专家
    - 角色塑造专家
    - 对话润色专家
    - 情节逻辑检查员
    - 文案改写助手
    - 多视角叙事专家
  - 每卡显示：头像、名称、描述、绑定模型标识
  - "+"按钮添加自定义角色
- **聊天区**：
  - 支持Markdown渲染
  - 流式打字机输出
  - 消息操作：复制、插入到编辑器、重新生成
- **输入区**：
  - 多行输入框（自适应，最大4行）
  - 发送/停止按钮
  - 附件：附加当前章节/选中文本/指定文档
  - 快捷指令`/`：续写、改写、扩写、缩写、翻译、起名

### 3.3 AI知识库检索功能（核心）

AI助手在回答和创作时，自动检索本地Obsidian知识库以获取上下文，保持创作一致性：

- **自动上下文注入**：
  - 当用户在某章节中请求AI辅助时，自动检索并注入：
    - 上一章节的完整内容或摘要
    - 当前章节已有内容
    - 当前章节中出现的角色卡信息（通过双链 `[[角色名]]` 自动识别）
    - 相关世界观设定（通过标签和链接关联）
    - 大纲中对应当前章节的剧情规划
  - 右侧面板"AI上下文"Tab实时显示AI检索到的内容片段

- **手动检索控制**：
  - 输入框旁的「知识库」按钮，点击弹出检索面板：
    - 搜索框：输入关键词搜索本地所有MD文件
    - 按类型筛选：章节/角色/世界观/大纲/全部
    - 搜索结果列表：勾选要注入给AI的内容片段
    - 已选上下文预览区：显示将发送给AI的上下文总量（token估算）
  - 支持拖拽文件树中的文件到输入框，直接作为AI上下文

- **智能关联推荐**：
  - 基于当前编辑文档的双向链接关系，自动推荐可能相关的文档
  - 推荐理由标签：「上一章」「关联角色」「相关设定」「大纲规划」
  - 用户可一键采纳或忽略推荐

- **上下文窗口管理**：
  - 显示当前上下文占用的token数量 / 模型最大token限制
  - 进度条可视化（超出时红色警告）
  - 自动优化：内容过长时自动摘要压缩，优先保留角色信息和近章内容

### 3.4 编辑器右键AI菜单
- 选中文本右键：AI续写、AI改写、AI扩写、发送给助手、生成描述、检查一致性

## 六、知识库系统（本地Obsidian式）

所有知识库数据以MD文件形式存储在用户本地指定目录中。

### 6.1 文件管理
- 树形浏览器
- 文件类型：`.md`章节、`.character.md`角色、`.world.md`世界观、`.outline.md`大纲
- YAML Front Matter元数据：
  ```yaml
  ---
  title: 第一章 初入山海
  type: chapter
  tags: [主线, 开篇]
  characters: [云溪, 山鬼]
  location: 云梦泽
  timeline: 第一年春
  created: 2024-01-01
  modified: 2024-01-02
  word_count: 3500
  status: draft
  prev_chapter: "[[序章]]"
  next_chapter: "[[第二章]]"
  ---
  ```
- 拖拽排序章节顺序

### 6.2 双向链接与图谱
- `[[文档名]]` 自动补全跳转
- `[[文档名#标题]]` 段落精确跳转
- `[[文档名|别名]]` 别名链接
- **关系图谱**：力导向图，节点大小=引用次数，颜色=类型，可缩放拖拽点击跳转

### 6.3 搜索（供用户和AI共用）
- 全文搜索（正则）
- 标签搜索
- 按类型/状态/时间筛选
- AI语义搜索（自然语言查询相关内容）
