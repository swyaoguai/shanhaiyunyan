## 维护要求

- 每次版本更新、Bug 修复、体验调整、测试或构建变更，都要在本文件补一条记录。
- 只记录新增了什么功能、修复了什么问题、调整了什么体验，以及测试或构建结果。
- 不写实现原理、排查过程、内部调用链或供应商底层细节；技术名词只保留到能识别功能或问题即可。
- 记录保持简洁，面向用户说明结果即可。
- 当前版本线从 `v1.0.0` 开始，后续发版按语义化版本递增。

## [v1.0.3] - 2026-05-20

### 修复

- 打包版后台日志改为按大小自动轮转并关闭逐请求访问日志，避免 `data/logs/agent.log` 长期运行后无限增长；诊断日志导出也限制为最近有限日志片段。
- API 配置测试会按所选 `api_type` 请求对应端点，`openai_responses` 不再误用 `/chat/completions`，而是测试 `/responses`。
- Agent 独立配置、复制配置和旧模型获取入口会保留并使用所选 `api_type`，避免单个 Agent 选择 Responses/Anthropic 配置后仍按 Chat Completions 调用。
- 统一处理调用方直接传入 `role=system` 消息的情况：Chat 会按所选格式保留或折入用户消息，Responses 会合并为 `instructions` 或 string `input`，避免不同业务入口请求格式不一致。
- 模型服务阻断等非重试错误会记录真实尝试次数，并减少 Communicator 意图检测兜底时的重复 traceback，便于定位实际模型配置问题。
- 探索仓 `*.tsc5.top` 配置调用稳定性提升，避免因请求头不兼容导致模型调用被拦截。
- 左侧底部“退出应用”按钮会正确遵守运行时开关，源码运行时不再因 CSS 覆盖 `hidden` 属性而误显示；若仍触发 409，会提示使用启动窗口关闭后台。
- 版本信息页、启动信息和运行时 API 改为读取统一的 `VERSION` 文件，避免发布到 `v1.0.2` 后界面仍显示 `v1.0`。
- Copilot 在章纲审阅断点收到“创作/继续”等确认语时，会续跑已确认任务池，不再重新生成世界观、角色和大纲。
- 创作合同确认时会把已有世界观、角色、大纲、章纲和正文标记为复用任务，避免任务预览显示“将复用”但正式执行又重新生成。
- 移除“浏览器窗口关闭后自动退出后台”机制，改为仅通过左侧底部“退出应用”操作关闭打包版后台进程，避免页面刷新、休眠恢复或浏览器后台回收误触发后端退出。

### 测试

- 后端启动、打包与版本信息验证通过：19 passed。
- 前端 Vitest 全量验证通过：143 passed。

### 构建

- 重新打包生成两个 `v1.0.3` Windows 安装包：无检索模型版约 60.83 MB，内含检索模型版约 102.13 MB。

- 封面生成会将平台目标尺寸映射为图像模型支持的请求尺寸，并在保存前本地裁切缩放到目标分辨率，避免将 800x600 直接传给不支持该尺寸的服务商接口。
- 封面生成结果支持点击查看大图和保存到本地，历史列表改用缩略图显示，并新增多选批量删除和及时保存提醒。
