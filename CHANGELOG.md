# 更新日志

## 维护要求

- 每次版本更新、Bug 修复、体验调整、测试或构建变更，都要在本文件补一条记录。
- 记录保持简洁，写清楚“改了什么”和“影响哪里”即可。
- 当前版本线从 `v1.0.0` 开始，后续发版按语义化版本递增。

## [未发布]

### 修复

- Copilot 智能判断模式会把“丰富/细化主角人设和世界观设定”识别为讨论型创作辅助，不再直接启动世界观生成或写入流程。
- Copilot 路由新增不依赖 ONNX 的结构化执行门控，将主题识别、操作类型和是否允许副作用分开判断，降低智能判断模式误执行风险。

### 构建

- 发布打包脚本改为只生成内含本地 ONNX 检索模型的安装包；无检索模型版仅保留为本地调试选项，不再作为正式发布产物。

## [v1.0.3] - 2026-05-20

### 修复

- 打包版后台日志改为按大小自动轮转并关闭逐请求访问日志，避免 `data/logs/agent.log` 长期运行后无限增长；诊断日志导出也限制为最近有限日志片段。
- API 配置测试会按所选 `api_type` 请求对应端点，`openai_responses` 不再误用 `/chat/completions`，而是测试 `/responses`。
- Agent 独立配置、复制配置和旧模型获取入口会保留并使用所选 `api_type`，避免单个 Agent 选择 Responses/Anthropic 配置后仍按 Chat Completions 调用。
- 统一处理调用方直接传入 `role=system` 消息的情况：Chat 会按所选格式保留或折入用户消息，Responses 会合并为 `instructions` 或 string `input`，避免不同业务入口请求格式不一致。
- 模型服务阻断等非重试错误会记录真实尝试次数，并减少 Communicator 意图检测兜底时的重复 traceback，便于定位实际模型配置问题。
- 探索仓 `*.tsc5.top` 的 OpenAI SDK 调用会覆盖默认 `User-Agent`；真实原因是 SDK 自带的 `AsyncOpenAI/Python` User-Agent 会被中转风控拦截，而同样请求用普通 `httpx` 或其它客户端可正常调用。
- 左侧底部“退出应用”按钮会正确遵守运行时开关，源码运行时不再因 CSS 覆盖 `hidden` 属性而误显示；若仍触发 409，会提示使用启动窗口关闭后台。
- 版本信息页、启动信息和运行时 API 改为读取统一的 `VERSION` 文件，避免发布到 `v1.0.2` 后界面仍显示 `v1.0`。
- Copilot 在章纲审阅断点收到“创作/继续”等确认语时，会续跑已确认任务池，不再重新生成世界观、角色和大纲。
- 创作合同确认时会把已有世界观、角色、大纲、章纲和正文标记为复用任务，避免任务预览显示“将复用”但正式执行又重新生成。
- 移除“浏览器窗口关闭后自动退出后台”机制，改为仅通过左侧底部“退出应用”操作关闭打包版后台进程，避免页面刷新、休眠恢复或浏览器后台回收误触发后端退出。

### 测试

- 后端启动、打包与版本信息验证通过：19 passed。
- 前端 Vitest 全量验证通过：143 passed。

### 构建

- 重新打包生成两个 `v1.0.3` Windows 安装包：无检索模型版约 60.83 MB，内含检索模型版约 102.13 MB。

## [v1.0.2] - 2026-05-20

### 新增

- 新增小说封面生成工作台，支持模板选择、项目元素提取、自定义元素、提示词预览、图像生成和项目封面历史保存。

### 变更

- 调整《小说封面生成功能设计方案》，明确世界观和时代背景只作为背景描述辅助信息，不作为封面生成必填核心。
- 封面生成工作台内置 13 套字体提示词与静态示例图，改为番茄平台 800x600 或自定义分辨率生成，并移除质量与端点模式选择。
- 封面提示词会从世界观、主角档案、物品设定和大纲中自动提取角色、场景、符号和情绪色彩，并在书名默认使用项目名时提示用户确认。
- 封面生成工作台将复制提示词和生成封面操作移动到提示词生成区域下方，补充图像模型使用说明，并在字体示例选中项右下角显示勾选标记。
- 收紧左侧资源栏与导航栏背景铺底，避免窗口边缘出现侧边栏缝隙，并修复底部“关于”按钮被遮挡的问题。
- 封面生成的模型选择改为只显示图片模型下拉选项，禁止手动输入语言模型，并在后端拦截明显的聊天模型误用。
- 封面描写提示词新增独立文本模型选择；项目资料和自定义元素为空时会先按字体模板补全，再可由文本模型润色。
- 调整封面生成表单字段顺序，让元素来源与平台分辨率并列，提示词和图片模型都按“API 配置 / 模型”成组排列。
- 封面生成新增“创作想法”输入，空项目时会结合书名、角色名和想法补全画面元素；文本模型被拦截时不再展示供应商原始错误，并移除重复的“元素描写”展示块。
- 封面生成会将平台目标尺寸映射为图像模型支持的请求尺寸，并在保存前本地裁切缩放到目标分辨率，避免将 800x600 直接传给不支持该尺寸的服务商接口。
- 封面生成结果支持点击查看大图和保存到本地，历史列表改用缩略图显示，并新增多选批量删除和及时保存提醒。

### 修复

- 安装器会在启动安装、正式写入文件和卸载前自动结束残留的 `山海·云烟.exe` 进程，避免旧后端进程占用文件导致更新无法继续。

### 构建

- 重新打包生成两个 `v1.0.2` Windows 安装包：无检索模型版约 60.82 MB，内含检索模型版约 102.10 MB。

### 测试

- 新增封面提示词构建、封面生成路由和前端封面工作台 DOM 测试。
- 封面生成相关验证通过：后端 pytest 15 passed，前端 Vitest 1 passed。

## [v1.0.1] - 2026-05-19

### 新增

- 导入小说后自动反向补全项目资料，包括角色卡、世界观、大纲、章节摘要、事件线和物品初稿。
- 多 Agent 协作链路新增标准 runtime event、JSONL 事件日志和 Dispatcher 生命周期事件，保留旧执行轨迹兼容字段。
- 多 Agent 协作事件新增 typed runtime message / artifact envelope，并在 Copilot 与协作状态流前端加入 renderer registry。
- RuntimeEventLog 新增分支摘要、压缩摘要和可转上下文包的轻量能力，Dispatcher 增加项目内可信 runtime hooks。
- 新增《小说封面生成功能设计方案》，规划封面提示词生成、项目元素提取、文生图、Responses 和 Chat Completions 兼容端点。

### 修复

- API 连接测试在内置探索仓API未配置时会提示先选择可用配置，避免直接测试占位配置。
- 修复自定义背景图重启后被全局覆盖层强模糊遮住的问题。
- 修复内含检索模型版首启未自动启用本地 ONNX 向量模型，导致知识库提示 provider 未配置的问题。
- 修复无限续写在选择非默认 API 配置时未保留 `api_type` 和 key 池，导致测试可用但实际创作端点不一致的问题。
- 无限续写会在全局激活配置变化后自动跟随新的默认配置，避免旧项目本地缓存继续请求过期的预设地址。
- 全局 API 设置页将内置配置显示为“探索仓API”，并把“打开中转站”快捷链接移动到该配置旁边。
- 发布打包改为读取 `VERSION` 文件并在构建前检测版本，支持 `--bump patch/minor/major` 自动升版，同时统一构建输出为 UTF-8，避免重定向日志乱码。
- Copilot 智能判断在用户询问设定推荐、方案或建议时会保持聊天讨论，不再误触发多 agent 创作任务池。
- 移除 Copilot 聊天入口的斜杠命令解析和前端快捷命令提示，统一改由自然语言意图判断处理。
- 资料库手动新增/编辑支持“自由内容”和“结构化模板”两种模式，并修复大纲、正文摘要等非 `name` 主字段类型误报“请输入名称”的问题。

### 构建

- 重新打包生成两个 `v1.0.1` Windows 安装包：无检索模型版约 59.07 MB，内含检索模型版约 100.37 MB。

### 测试

- 后端 pytest 全量覆盖：869 passed，9 skipped。
- 前端 Vitest 全量覆盖：19 passed + 137 passed。
